## 状态说明

| status      | 含义          | 发布动作                 |
| ----------- | ----------- | -------------------- |
| `draft`     | 初稿 / 审阅中    | 不发布                  |
| `reviewed`  | 首次审阅通过      | CREATE               |
| `published` | 已上线，本地与线上一致 | 无                    |
| `revised`   | 已上线但本地又改    | UPDATE（同 wp_post_id） |
| `archived`  | 下线或废弃       | 不发布                  |

### 三、M02 使用须知与限制

开篇即提醒：MDStudio 是**在线冒烟/装输入**工具，不是无限算力集群。数字以站点实际上线为准，写作用户体感并链 VIP 购买页；**勿写** `config.json` / entitlements 键名。依据 `compute_limits.md`、`vip_gate.py`、部署说明中的用户表述。

建议表格与清单：

| 类别 | 用户侧写什么（示例口径，定稿对产品） |
|------|--------------------------------------|
| **账号与权益** | 免费大概能：登录、仓库浏览/预览…；VIP 才能：绘制、力场、装盒、测试、仓库导入…（按现状） |
| **单分子规模** | 单次力场/结构约原子数上限（如 ~1000） |
| **装盒规模** | Packmol 在线阈值原子数阈值；过大时改为下载 `run_packmol.sh`、本地跑完再上传 `simbox.xyz` |
| **测试模拟** | 仅短冒烟；原子数/墙钟时间上限；**不能**当生产 MD |
| **上传与空间** | 单文件大小、工作区总容量 |
| **任务与排队** | 超时、可取消、失败时日志看什么 |
| **格式与配对** | LigParGen 要 gro/pdb+itp 成对；水/离子模型要同系列；周期性 CIF vs 团簇 |
| **禁止 / 慎用** | 勿上传未发表涉密数据到公共环境；大轨迹勿指望在线存；生产长跑请下栽到本地/集群（链 T09/T10） |
| **翻车对照** | 超时红了 / 原子超限 / 无 VIP 点开 Tab / pack 覆盖丢修改 → 怎么办 |

文末：建议：**M01 练通后立刻读本文**；再进 M03 地图 / Tab / 配方。

### 二、环境怎么处理：`myenv` 与 `mymlenv`

#### 1. 结论（推荐）

**两个 Conda 环境，互不合并。**

| 环境名 | 定位 | 典型内容 | 谁维护 |
|--------|------|----------|--------|
| **`myenv`** | MD 分析主环境（已有） | MDAnalysis、ASE、NumPy/Pandas/Matplotlib、Jupyter；可轻量加 sklearn | [T01](在线资源/01-技术文档/T01-分子模拟工作平台搭建.md) + `downloads/myenv.yml` |
| **`mymlenv`** | ML / DL 扩展环境（新建） | 在 MD 分析底座上加 **scikit-learn、PyTorch、RDKit** 等；可选 GPU 版 torch | 本规划 T 线「平台篇」+ `downloads/mymlenv.yml` |

> **命名说明：** 仓库现有草稿名为 `myenv-ml`（见 `downloads/myenv-ml.yml`、T30）。对外统一改为 **`mymlenv`**，避免与 `myenv` 混淆；定稿时把 yml 的 `name:`、zip、文中字符串一并改齐。进阶 ML 势可另开 **`mymlips`**（或沿用文中的 `myenv-mlip`），不与 `mymlenv` 混装。

#### 2. 为什么不装进同一个 `myenv`？

- PyTorch / CUDA、RDKit、部分 GNN 依赖**体积大、版本敏感**，易拖慢 MD 日常环境更新。  
- CPU 本机与 GPU 集群的 torch 安装命令不同，分环境可在项目 README 写清「本机 CPU / 集群 CUDA」。  
- 读者只做 RDF/出图时不必背 ML 依赖。

#### 3. 怎么用（读者侧一句话）

```text
日常 MD 分析、画图、写脚本  →  conda activate myenv
描述符 / sklearn / PyTorch / RDKit →  conda activate mymlenv
训练 ML 势、重型框架          →  mymlips（另文）+ 集群 GPU
```

Jupyter / VSCode：**各环境注册独立 kernel**（如 `Python (mymlenv)`），Notebook 里选对核即可；同一项目可两个 kernel 切换，轨迹仍放在同一 `data/` 目录。

#### 4. 技术实现清单（写作时落实）

| 项 | 做法 |
|----|------|
| 源文件 | `downloads/mymlenv.yml`（由现 `myenv-ml.yml` 改名/改 `name`） |
| 分发包 | `mymlenv.zip`（仅含 yml），上传 WP 媒体库，**免费**链出（同 myenv 策略） |
| 入口文 | T 线「ML 工作平台」或扩写 T30：一键 `mamba env create -f mymlenv.yml` |
| 与 T01 关系 | T01 **只**讲 `myenv`；文末「若做 ML → 见 ML 平台 / T30」，不在 T01 里塞 torch |
| GPU | 文档写清：yml 默认 CPU torch；需 GPU 时按 [pytorch.org](https://pytorch.org) 在已激活 `mymlenv` 中重装，并记入项目 README |
| 集群 | 节点上 `module`/`conda` 加载同一 `mymlenv` 或容器镜像；训练作业用 SLURM（链 T10） |
