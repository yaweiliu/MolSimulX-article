## 1. 写给谁、写成什么样

| 原则 | 说明 |
|------|------|
| **读者** | 分子模拟 / ML 方向的科研学习者：会一点 Python，不一定熟 Linux、集群、Git |
| **语气** | 通俗、口语化，像师兄带师弟——少堆术语，多讲「这一步解决什么问题」 |
| **结构** | 先场景、再步骤、再表格小结；长文用 `## 一、二、三、` 分节 |
| **篇幅** | 单篇讲清一个主题；细节拆到专题文，总览文只串链路 |

**导语（开篇 / 前言）应包含：**

1. 这篇解决什么痛点（用具体场景，如「轨迹几 GB、Git push 卡死」）  
2. 本文讲什么、**不讲什么**（与姊妹篇分工）  
3. 前置条件（如「先配好 SSH」「假设已有 myenv」）

### 1.1 导语不能过于简单（尤其 VIP）

导语是读者决定「往下读 / 开通会员」的第一屏，**禁止**写成两三句干巴巴的定义 + 「本文讲 X」。要有吸引力：先抛场景或痛点，再亮出这篇能带走什么，必要时用一小表点名关键概念或应用（如输运系数对上电池、热管理）。

| 要求 | 说明 |
|------|------|
| **厚度** | 一般 **3–6 段**（可含一张小表）；不要只有「X 是什么 + 本文讲 X」两段交差 |
| **钩子** | 开头用具体场景、反差或「你会卡住的问题」，少用「本文系统介绍……」 |
| **地图感** | 让读者立刻感到「这张谱系 / 这条流程和我有关」——点名关键量、适用场合或与姊妹篇分工 |
| **VIP 更严** | `paywall: vip` 时，**付费墙外的导语几乎是整篇的免费门面**：必须写够吸引力与信息密度，让人愿意往下读 / 开通；勿把干货全塞进墙内、墙外只留空壳定义 |
| **仍要克制** | 吸引 ≠ 标题党；不剧透墙内逐步操作的全部细节，但墙外要说清「学完能解决啥、和谁对照」 |

**偏弱（避免）：**

> 输运系数描述体系对扰动的响应。本文介绍扩散、粘度与热导，以及平衡与非平衡算法。

**更好（示意）：**

> 密度、RDF 回答「平衡在哪」；电池电解液「离子跑多快、内阻大概怎样」、冷却液「导热好不好」，问的是另一类量——扩散、电导、粘度、热导。先认清谱系里有谁、对上哪些应用，再选平衡涨落还是非平衡驱动……

VIP 结构仍见 [发布与上线](发布与上线.md)「付费与 VIP」：导语留在墙外免费；墙外导语按上表写厚，不要为了「省字」把前言砍成定义句。

**篇末建议有：**

- **小结**（3–5 条可执行的要点）  
- **前置阅读 / 下一步**（用文章标题链出，见第 3 节）  
- 可选 **常见问题**（FAQ）

## 2. 通俗化写法

语气要好懂，但**不要用元标注告诉读者「下面要讲人话了」**。直接写清楚即可。

### 2.1 禁止的元话头（直接删掉，改写成正文）

| 避免（正文 / 标题 / 表头） | 说明 |
|---------------------------|------|
| 「人话」「读成人话」「讲人话」 | 像在嘲讽读者；删掉标签，把解释写进正文 |
| 「通俗地讲」「通俗讲」「大白话」 | 同上 |
| 「白话：」「白话」作表头 / 小节标题 | 表头改用「含义」「说明」「直观理解」等 |
| 「简单说：」「说白了：」当段首标签 | 可保留「也就是说」等自然过渡，不要当「降智开关」 |
| 「认门牌」「先认门牌」 | 像在暗示「只背名字不用懂」；改成「常见名字 / 常见做法 / 先分清种类」等直接说法 |
| 「选型」作标题、表头或套话（「选型总闸」「选型梯子」「选型速查」） | 改成「怎么选」「如何判断用哪条路」「适用场合」；动词可用「选择」「判断」 |

**对：** 直接写「关联长度可以想成……」  
**错：** 「关联长度是啥？（人话）」「白话：关联长度就是……」

**对：** 「常见热浴做法见下表」「按问题判断用经典力场还是 AIMD」  
**错：** 「常见热浴（认门牌）」「力场选型总闸」

### 2.2 用场景和比喻，不用空话

| 避免 | 改为 |
|------|------|
| 「本文系统介绍 X 的配置与最佳实践」 | 「连 Git、登集群、传轨迹都离不开 SSH；配好一次密钥，以后不用一遍遍敲长域名」 |
| 「版本控制工具」 | 「项目的时光机，避免 `分析_最终版_v3.ipynb`」 |
| 「交互式开发环境」 | 「浏览器里边写边跑的 Notebook 工作台」 |

比喻要贴切、一两句即可，不要通篇比喻。

### 2.3 表格承担「对照」和「清单」

- 组件对照（工具 → 含义 / 直观理解 → 详细教程）  
- 适合 / 不适合（如 Notebook 适合探索、不适合几天几夜生产跑）  
- 排错速查（现象 → 可能原因 → 怎么办）

### 2.4 Tips 块

非显而易见、但省时间的点，用引用块标出：

```markdown
> **Tips：** 登录节点能否访问外网装包，各校不同——装之前先问管理员。
```

可选前缀：`Tips`、`可选`、`注意`。不要滥用，一篇 3–6 处为宜。

### 2.5 命令与代码

- 命令注明**在哪跑**（本机 WSL / 登录节点 / 计算节点）  
- 长命令可拆步，关键参数用行内注释说明  
- 复制即用的片段优先；理论推导留给知识文档（K 系列）

### 2.6 姊妹篇分工要说清

同一主题拆成「工具操作」与「科研规范」时，开篇对照表写明白，避免读者不知道先读哪篇。

**示例（Jupyter）：**

| 主题 | 读哪篇 |
|------|--------|
| 安装、界面、单元格、快捷键 | [JupyterLab简明教程](在线资源/01-技术文档/T11-JupyterLab简明教程.md) |
| 目录、复现、Git、清输出 | [Jupyter Notebook科研使用规范](在线资源/01-技术文档/T16-Jupyter Notebook科研使用规范.md) |

## 3. 引用与互链（不要编号）

### 3.1 读者可见文字：禁止 T## / E## / K##

**正文、表格、链接锚文字**里不要出现文章编号，一律用**完整标题**或自然语言。

| 错误 | 正确 |
|------|------|
| 见 [T10](T10-集群与SLURM简明教程.md) | 见 [集群与SLURM简明教程](T10-集群与SLURM简明教程.md) |
| 见 T01 第七节 | 见 [分子模拟工作平台搭建](T01-分子模拟工作平台搭建.md) 第七节 |
| 跟一篇 **T21 绘图** 练手 | 跟一篇 [NumPy与Matplotlib简明教程](T21-NumPy与Matplotlib简明教程.md) 练手 |
| 工作流 T12–T18 | 工作流与写作（目录规范、数据备份、论文图等） |
| K01–K31 概念文 | [知识文档](在线资源/00-知识文档/README.md) 栏目下的概念文 |

**允许出现编号的地方（读者一般看不到或属内部管理）：**

- YAML 字段 `id: T01`  
- **文件名** `T10-集群与SLURM简明教程.md`（链接目标路径）  
- [内容规划](内容规划.md)、编辑用总表  
- [资源导航.md](在线资源/资源导航.md) 等内部导航表中的 `id` 列（维护用，正文仍用标题链）

### 3.2 链接锚文字 = 文章标题

```markdown
<!-- 好 -->
[SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md)
[NumPy与Matplotlib简明教程](../01-技术文档/T21-NumPy与Matplotlib简明教程.md)

<!-- 差 -->
[SSH](T08-SSH密钥与config配置简明教程.md)
[远程连接集群](T07-VSCode与Cursor远程连接集群.md)   <!-- 应写完整标题 -->
[T01 第二节](T01-分子模拟工作平台搭建.md)           <!-- 应写标题 + 节名 -->
```

节号可以跟在标题后：**「[分子模拟工作平台搭建](…) 第二节」**，不要用编号代替标题。

### 3.3 本地路径约定

| 场景 | 写法 |
|------|------|
| 同目录技术文互链 | `[Git简明使用教程](T04-Git简明使用教程.md)` |
| 跨栏目（技术 → 案例） | `[NumPy与Matplotlib简明教程](../01-技术文档/T21-NumPy与Matplotlib简明教程.md)` |
| 技术 → 知识文档 | `[分子动力学模拟概述](../00-知识文档/K02-分子动力学模拟概述.md)` |
| 技术 → 资源导航 | `[资源导航](../资源导航.md)` |

- 源稿里**不要写 WordPress 绝对 URL**；发布时由 `publish.py` 替换。  
- 未发布文章在 Obsidian 里仍可链 `.md`；线上未发布链显示为待发布（见 [发布与上线](发布与上线.md)）。

### 3.4 尚未成稿的选题

无对应 `.md` 文件时，**不要做空链**。可写纯文字并注明状态：

```markdown
- Docker与Apptainer入门（规划中）—— 环境复现的另一条路
```

### 3.5 站内脉络 vs 单篇阅读

- **单篇文末**：「前置阅读 / 下一步」用标题链接即可。  
- **全站顺序、编号块**：只放在 [内容规划](内容规划.md)，**不要**在公开正文中写「按 T01→T11 顺序读完」。

## 4. 文首元数据与系列标签

### 4.1 YAML（真源）

```yaml
---
id: T01
title: 分子模拟工作平台搭建
series: 在线资源
tier: 01-技术文档
status: draft
topic: 平台搭建
paywall: free
---
```

知识/技术文 YAML 里 `paywall` 现用：`free` · `vip` · `download-vip` · `download-paid`（旧 `none` 等同 `free`）。详见 [发布与上线](发布与上线.md)「paywall」。

- **`id` / `title`**：`id` 供脚本与内部表；`title` 即对外标题，也是互链时用的名称。  
- **`topic`**：站点分类下的主题词，与 `系列标签` 第三、四格呼应即可。

### 4.2 系列标签（正文第一行引用块）

紧跟 YAML 后写一行，**固定 4 个**反引号标签，用 ` · ` 分隔：

```markdown
> **系列标签：** `技术文档` · `平台搭建` · `MolSimulX` · `环境配置`
```

| 栏目 | 第 1 格 | 第 2–4 格示例 |
|------|---------|----------------|
| 01 技术 | `技术文档` | 主题词，如 `SSH`、`工作流`、`科研绘图` |
| 00 知识 | `知识文档` | `分子模拟`、`力场`、`MolSimulX` 等（共 4 格） |
| 文件名 | `K02-分子动力学模拟概述.md` | 与 `T##-` / `C##-` 一致；正文互链只用标题 |
| 02 实战 | `实战案例` | 体系/场景名（如 `LJ流体`、`水盒子`）；勿用纯工具名当主标签 |

标签要**能检索、能概括**，不要凑字数；同一主题姊妹篇标签要有区分度（如 `JupyterLab` vs `可复现`）。

## 5. 章节与排版习惯

- **一级结构**：`## 一、……`、`## 二、……`（中文序号 + 顿号）  
- **小节**：`###` 即可，不必再写「1.1」除非表格需要  
- **分隔**：大节之间用 `---`  
- **引用块里不要嵌套围栏代码**：`>` 引用中写 bash 等围栏代码块时，多数 Markdown / WP 渲染会错乱（出现 `"bash`、代码块样式异常）。做法：引用只写说明，**代码块放在引用外**的普通段落里。  
- **加粗与全角括号（Obsidian）**：`**…）**` 紧贴后面的汉字时，预览常把 `**` 显示成字面量（如 `有限元（FEM）**等`）。写法二选一：  
  - 推荐：`**有限元**（FEM）等`（加粗不含全角括号）  
  - 或：`**有限元（FEM）** 等`（闭 `**` 后留空格再接下文）  
  同理避免 `）**字`、`字**（` 粘连导致的边界错误。  
- **图片**：Obsidian 可用 `![[name.png]]`；发布前由 `sync_article_images.py` 归档为 `images/articles/.../web/*.webp`  
- **封面图与正文**：**导语（前言）不要解释封面图**——图放在导语之后、第一节之前即可，下面不要跟「上图是……」「封面画的是……」这类说明。若图与正文某节直接相关，在**正文该节**里自然带一句（如「封面左图……」「与封面上的烧瓶对照……」）；只是装饰、或和文意对不上，**不要硬写**——在协作里标明「封面图不太贴」，请作者更换。  
- **封面显示**：发布时正文**第一张图**会标为封面；封面与正文插图在站点上均为 **`max-height: 420px`**、`object-fit: contain`（比例自由、不裁切、水平居中）。源图像素与比例不必统一；优先横图。  
- **外链**：官方文档、ColorBrewer 等用 `https://` 完整 URL  
- **付费/VIP**：结构见 [发布与上线](发布与上线.md)「付费与 VIP」，此处不重复

## 6. 审阅自检清单

发布前或 PR 前过一遍：

- [ ] 导语是否说清「写给谁、解决啥、前置啥」；是否**够厚、有场景钩子**（忌两三句空壳定义）  
- [ ] **VIP 篇**：墙外导语是否已有吸引力与信息密度（勿只留定义句等开通）  
- [ ] 语气是否口语化，有没有未解释的首字母缩写  
- [ ] 是否避开「人话 / 通俗地讲 / 认门牌 / 选型」等禁止套话（见 §2.1）  
- [ ] 封面图是否与主题相关；**导语下无「上图/封面」解说**；需要时是否在正文结合封面描述；不相关是否已提示更换  
- [ ] 加粗是否避开 `）**汉字`（应用 `**术语**（英文）` 或闭 `**` 后空格）  
- [ ] 互链是否用**完整标题**；跨目录路径是否正确  
- [ ] 系列标签是否 4 个、与 `topic` 一致  
- [ ] 与姊妹篇分工是否写清（避免 T11/T16 类重复）  
- [ ] 篇末是否有小结 + 前置/下一步  
- [ ] `status` 是否与审阅阶段一致（见 [发布与上线](发布与上线.md)）

### 可选：批量检查编号残留

在 `在线资源/01-技术文档/` 下对 T01–T18 等跑检查（排除 YAML `id:` 与链接文件名）：

```bash
cd 在线资源/01-技术文档
python3 << 'PY'
import re
from pathlib import Path
pat = re.compile(r'(?<![/\w-])\b[TEK]\d{2}\b(?![\w-])')
for f in sorted(Path('.').glob('T*.md')):
    for i, line in enumerate(f.read_text().splitlines(), 1):
        if line.strip().startswith('id:'): continue
        if pat.search(line) and not ('](' in line and pat.search(line.split('](')[-1] or '')):
            print(f'{f.name}:{i}: {line.strip()[:100]}')
PY
```

有输出则需改写成标题或自然语言。
