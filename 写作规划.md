## 总方向：三大块怎么铺

站点「在线资源」继续按 **知识（K）→ 技术（T）→ 实战（C）** 推进；在线工具（M）与解决方案（S）另线配合。下面三条是**后续部署的主轴**。

```
知识文档（K）  →  把「算什么、为什么」讲深：分子模拟进阶 + ML 基础 + ML×模拟
技术文档（T）  →  把「用什么工具」讲清：Python 基础库 + 模拟相关库/工具简明教程
实战案例（C）  →  把「怎么做完一件事」跑通：MDStudio/工具建模 → 简单方法 → 大型项目
```

## 1. 知识文档（K）：进阶理论与方法地图

在现有 K01–K31 主线上，**继续加深、补齐**，而不是重复「开跑入门」：

| 方向 | 写什么 | 举例 / 锚点 |
|------|--------|-------------|
| **分子模拟进阶** | 统计力学、采样、系综与算法边界上的加深；经典教材与专著脉络 | 以 Daan Frenkel & Berend Smit《Understanding Molecular Simulation》等为线索，补 MC、自由能、稀有事件、有限尺寸等进阶篇（可与现有 K14 / K17–K24 等互链，避免重复） |
| **机器学习基础** | 面向模拟读者的 ML 通识，不写成纯 CS 课 | 数据与特征、经典机器学习、神经网络与深度学习基础（现有 K28 / K29 可继续展开成小系列） |
| **机器学习 × 分子模拟** | 两者交汇处的概念与方法选型 | 机器学习力场（MLFF / ML 势）、表示学习与描述符、主动学习采点、性质预测与生成模型等（与 K05、后续 ML 实战互链） |

**边界：** 知识文讲清概念、判据、适用场合与常见坑；**不**用整篇代替库语法课（归 T），**不**用整篇代替「从建模到出图」故事线（归 C）。

## 2. 技术文档（T）：库与工具的简明教程

在平台 / 集群 / 工作流（T01–T20 等）之外，**工具速成线继续扩容**：

| 方向 | 写什么 | 举例 |
|------|--------|------|
| **基础 Python 库** | 读文档够用、照抄能跑的短教程 | NumPy、Matplotlib、pandas、（按需）SciPy 等（已有部分在 T21 一带，可补缺） |
| **分子模拟相关库** | 结构 / 轨迹 / 分析常用栈 | MDAnalysis、ASE、（按需）MDTraj、pymatgen 等 |
| **领域专用工具** | 某一类建模或前处理的短教程 | 如 **pyCOFbuilder** 等；一篇讲清安装、最小例子、与 LAMMPS / 本站工作流怎么接 |

**边界：** 技术文是「工具说明书 + 最小可运行例子」；**不**展开长故事线案例（归 C），**不**替代力场 / 系综等原理长文（归 K）。

## 3. 实战案例（C）：建模 → 方法 → 项目

实战必须**结合分子模拟场景**，按体量分三层往上铺：

| 层级 | 写什么 | 举例 |
|------|--------|------|
| **模型搭建** | 结合 **MDStudio** 与本地工具，把某类体系搭到可跑 | 受限溶液（板—溶液—板）、异质墙、孔道 / 夹层等；交叉链接 M 系列功能文 |
| **简单模拟方法实现** | 一条可复现短链路：输入 → 短算 → 分析出图 | 机械控压、短平衡 + 密度 / 应力剖面、基础 NEMD 示意等 |
| **大型项目实现** | 多步骤、多文件、接近「小课题」的完整项目 | 较长生产 + 系统分析、MLFF 训练 / 验证、CG 映射与混合体系等；资源包宜 `download-vip` / `download-paid` |

**边界：** 实战讲「这一次具体怎么做完」；原理深挖链回 K，库语法链回 T，MDStudio 操作链回 M。
