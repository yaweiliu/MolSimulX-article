# 在线工具（MDStudio）文章规划

> **状态：** 规划稿（内部）  
> **位置：** `MolSimulX-article/` 根目录  
> **栏目：** `在线工具/` · 编号前缀 **`M`**（MDStudio；**不再使用 W**）  
> **依据：** [MolSimulX-core/wiki/README.md](../MolSimulX-core/wiki/README.md) 中 **Web 界面 — MDStudio** 与相关用户向 wiki（力场转换、分子仓库、水模型、CIF、计算限制等）  
> **写法：** 教程 + 功能说明；**少写** CLI 模块名、API 分层、部署/SSO 实现、势函数推导、`.ff` 格式细则（那些留在 core wiki / 给开发者）

产品入口：`mdstudio.molsimulx.com`（本地联调见 core 文档，**本栏目正文不展开部署**）。

## 一、原则

| 要写 | 不要写（或仅一句带过） |
|------|------------------------|
| 哪个标签页、点哪里、选什么、得到什么文件 | FastAPI / htmx / 子进程编排 |
| 推荐操作路径（从结构到 `data.lmp`/`in.lmp`） | `msxflow ff` 命令行全参数表 |
| 用户能感知的限制（原子数、上传大小、任务超时、VIP） | `config.json` 键名与运维改法 |
| 电荷/力场选项「适合什么场景」 | GAFF 匹配 tier、`[guess]`、单位 1/2 推导 |
| 失败时日志里看什么、常见翻车 | 源码路径、测试文件名 |

**与在线资源分工：** 概念（力场、系综）→ 知识文档；本机装 LAMMPS/Conda → 技术文档；浏览器里完成「结构→输入」→ **本栏目**；完整科研故事 → 实战案例。

### B. 按标签页（功能教程）

与界面 Tab 顺序对齐（力场转换 → 孤立分子 → 周期分子 → 分子仓库 → 力场生成 → 超胞变换 → 搭建盒子 → 测试模拟）：

| id | 标题（草案） | 大概写什么 | 对应 Tab / 素材 |
|----|--------------|------------|-----------------|
| **M03** | 力场转换：LigParGen / 外部力场导入 | 何时用；准备 `.gro`+`.itp`；上传后得到什么；如何接到「搭建盒子」；常见失败（文件不配对） | 力场转换；ligpargen_convert（操作摘要） |
| **M04** | 绘制孤立分子（Ketcher） | 打开绘制；画完导出 3D；拷贝 SMILES；送入力场生成；原子数过大时提示 | 孤立分子 |
| **M05** | 构建周期结构 | 六类模板各一节短说明：晶体 / 表面 / MX2 / 纳米管 / 石墨烯带 / 团簇；参数含义用「要什么结构」语言；输出 CIF；再去力场生成 | 周期分子 |
| **M06** | 分子仓库：浏览、预览与导入 | 分类与搜索；点名称预览；导入到当前目录；与自建文件的关系；示例：离子液体单体、水模型入口 | 分子仓库 |
| **M07** | 力场生成（从结构到 `.ff`） | 选文件或 SMILES；力场类型（GAFF/GAFF2/Dreiding）怎么选（场景表）；电荷方法怎么选（MMFF94 / AM1-BCC / RESP / EQeq）——**只写适用场景与耗时体感**；产物 `{name}_ff.xyz` + `{name}.ff`；RESP 断点续跑（上传结果文件）点到为止 | 力场生成 |
| **M08** | 超胞与几何变换 | 何时需要扩胞/平移/旋转；选 xyz；nx/ny/nz 与正交化；看 `*_trans.xyz` | 超胞变换 |
| **M09** | 搭建模拟盒子 | 三步：① 生成 pack.inp → ② Packmol 装盒 → ③ 生成 LAMMPS；多组分怎么加；盒子尺寸/密度；周期方向；产物 `simbox.xyz`、`data.lmp`、`in.lmp`；可终止任务 | 搭建盒子；msxbox 产出 |
| **M10** | 测试模拟（冒烟） | 对工作区 `in.lmp` 短跑；能验证什么、不能代替正式生产；原子数/时间限制提示 | 测试模拟 |

## 五、每篇统一结构（写作清单）

1. **你要完成什么**（一句话目标）  
2. **打开哪里**（URL / 登录状态）  
3. **步骤**（编号 + 界面用语，少术语）  
4. **你会得到哪些文件**（文件名表）  
5. **接着做什么**（链下一篇 M 或实战案例）  
6. **常见问题**（3–5 条）  
7. **相关**（知识文档概念链；勿链部署 wiki）

篇幅：功能篇宜短（一屏流程 + 一张主图）；闭环篇 M13/M14 可稍长，仍避免后台细节。
