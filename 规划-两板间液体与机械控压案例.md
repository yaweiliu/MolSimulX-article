## 二、为什么都归「实战案例」，而不是 MDStudio

1. **栏目边界已定**：MDStudio（M）= 浏览器工具**功能说明**；`实战案例`（C）= **结合分子模拟的完整 / 半完整场景**（建模→跑 Lammps→分析→出图）。见 [内容规划.md](内容规划.md) 栏目边界表。
2. **配方已从 MDStudio 移除**：原 M13–M18 六篇「配方」已删除，方向就是把「特定体系怎么搭」的案例统一收进实战案例，避免 MDStudio 再背场景内容。
3. **这两个案例超出 MDStudio UI 能力**：
   - 两板间液体：需要 ASE 建**石墨烯墙**、`wall.ff`、把液体**夹在两墙之间**的**自定义 packmol 脚本**（`pack.inp` / `pack_mod.inp`），并区分 **homo / hetero 墙**——MDStudio 的「搭建盒子」是常规周期盒装填，不覆盖受限约束。
   - 机械控压：纯 **LAMMPS 技巧**（活塞 / 墙受力控局部压强 + 应力剖面分析），与 Web 工具无关。
4. **需本地 LAMMPS + Notebook**：两篇都要本地跑 Lammps、用 Notebook 出图，属实战案例的标准形态。

> **搭模型的复用策略（关键）**：正文用「MDStudio 能做的先在 MDStudio 做」的口吻组织——各组分 `.ff` 用 [M09 力场生成] 得到；常规装盒思路链 [M11 搭建盒子]；手改 `pack.inp` 与回传链 [M04 资源管理器]。**受限特有步骤**（墙、夹层 pack.inp、homo/hetero）明确标注「当前需本地 fftool + packmol 完成，MDStudio 暂不支持」。既不重复 MDStudio，又把工具引流进案例。

## 三、两篇文章安排

### 文章 A：两板间液体——受限体系建模与开跑

- **对应上位选题**：[内容规划.md](内容规划.md) 的 **C12「两墙夹水」**（主线）/ **C16「石墨烯夹层中的离子液体」**（进阶变体）。
- **素材**：`example_il_between_two_graphenes`（该 notebook **同时含** 2000 个 SPC/E **水**分子路径 **与** IL(c4c1im/PF6)+ACN 路径；石墨烯墙用 `ase.build.graphene_nanoribbon` 生成）。
- **一句话**：用 ASE 造两片石墨烯墙 → 自定义 packmol 把液体夹在两墙之间 → 生成 `data.lmp` / `in.lmp` → 短平衡 + 密度剖面示意。
- **免费门面**：建墙、装盒思路、关键 `pack.inp` 片段说明、初始构型图（`init_box.png`）、homo/hetero 墙示意（`homo_wall.png` / `hetero_wall.png`）。
- **VIP / 资源包**：完整 `pack.inp`、`wall.ff`/各组分 `.ff`、建模 notebook、可跑 `in.lmp` + 精简 `data.lmp`。
- **前置**：[ASE结构构建入门](在线资源/01-技术文档/T23-ASE结构构建入门.md)（T23）、K07（边界 / 墙）、K03（水模型）；建模复用链 M09 / M11 / M04。

> **待你拍板：水 vs 离子液体。** 你说的是「水分子在两个板间」，但素材主体是 IL+ACN（同一 notebook 也保留了 SPC/E 水路径）。三种落地取向：
> 1. **主写「两墙夹水」（推荐先行）**：SPC/E 水更普适、门槛低，直接充当 C12；石墨烯夹层 IL 作为后续 C16 变体。
> 2. **主写「石墨烯夹层 IL」**：贴合素材，直接做 C16，水作为「换成水模型」的变体一段。
> 3. **一篇讲通法 + 两个变体**：正文用通用「液体夹两板」框架，水与 IL 各给一节（篇幅偏长，可上下篇）。

### 文章 B：LAMMPS 机械控压（Mechanical barostat）

- **对应上位选题**：[内容规划.md](内容规划.md) 的 **C13「机械式控压」**。
- **素材**：`example_mechanical_barostat`（`in.lmp` / `input.lmp` / `_functions.py` / `result_*` / notebook / `md_trj.gif`）。
- **一句话**：界面 / 多相体系里压强不均匀、全局压浴失效时，用**机械方式**（活塞 / 墙受力）控局部压强；读懂输入 → 跑短算例 → 解析应力剖面 → 与 NPT 压浴对比（概念）。
- **免费门面**：为什么全局 barostat 在界面体系失效、机械控压思想、`in.lmp` 关键片段、`md_trj.gif` 与应力剖面图。
- **VIP / 资源包**：完整 `in.lmp` / `input.lmp`、`_functions.py`、分析 notebook、短样例输出。
- **前置**：文章 A（或等价受限体系）、K11（控温控压）；可链 K19（压强 / 表面张力）、K22（NEMD）。
- **文献锚点**（素材里已给，正文可引）：J. Chem. Phys. 2018, 148, 064706；及课题组既往工作。
