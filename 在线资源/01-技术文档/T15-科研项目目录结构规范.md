> **系列标签：** `技术文档` · `工作流` · `项目规范` · `Git`

做模拟久了，桌面容易堆满 `分析_最终版_v3.ipynb`、几 GB 的 `dump.lammpstrj`，还有说不清哪来的 `in(1).txt`。半年后你要复现、写论文、换电脑跑——**找不到文件比跑不动模拟更折磨人**。

可以把项目根目录想成一间**工作室**：左边抽屉放输入和配置，中间工作台写脚本和 Notebook，右边大柜子存轨迹（别往 Git 里塞），墙上贴运行日志。本文给出 MolSimulX 推荐的分子模拟项目布局，**复制目录树就能开新项目**。

学完能带走：一套 MolSimulX 推荐目录树（`config/`、`lammps/`、`data/raw/`、`figures/` 各放啥）、README 和 `.gitignore` 该写什么，以及相对路径怎么写才在本机和集群都不挂。小文件版本历史用 [Git简明使用教程](T04-Git简明使用教程.md)；轨迹放哪、跑完记啥元数据、冷备份怎么做，跟 [数据管理与备份](T17-数据管理与备份.md) 配套读。

## 一、为啥要折腾目录？不规整会怎样

| 常见惨状 | 规整之后 |
|----------|----------|
| Notebook 文件名全是 `final`、`真的最终` | `notebooks/01_`、`02_` 固定顺序，脚本在 `scripts/` |
| 轨迹 `git push` 一把，仓库 2 GB，队友骂娘 | 大文件进 `data/raw/`，`.gitignore` 挡在门外 |
| 论文 Methods 要写力场、种子，全忘了 | `config/` + `docs/runlog.md` 每次跑完记一笔 |
| 笔记本路径 `C:\Users\...`，集群上脚本全挂 | 项目内**相对路径**，根目录名统一 |

和 [Git简明使用教程](T04-Git简明使用教程.md)、[数据管理与备份](T17-数据管理与备份.md) 一起看，效果最好：Git 管「该管的」，备份管「大文件」。

> **Tips：** 目录规范不是形式主义——是为了 **README 能写清复现步骤**、审稿人要数据时你能在一小时内找到对应 run。

## 二、推荐目录树（单课题、单 Git 仓库）

上图四色对应下面四类；完整树如下（`#` 后是说明）：

```
my-md-project/                 # 项目根 = Git 仓库根
├── README.md                  # 新人/半年后的你：怎么复现
├── .gitignore                 # 告诉 Git：轨迹别进来
├── environment.yml            # Conda 环境（或注明用平台 myenv）
│
├── config/                    # 🔵 输入与配置（少改）
│   ├── forcefield/            # 力场参数等小文件
│   └── params.yaml            # 温度、步长、系综（可选）
│
├── structures/                # 🔵 初始结构
│   ├── system.data
│   └── README.md              # 结构从哪来、怎么建的
│
├── lammps/                    # 🔵 模拟输入 + 交作业脚本
│   ├── in.minimize
│   ├── in.npt
│   └── job.slurm
│
├── scripts/                   # 🟢 可重复跑的 .py / .sh
│   ├── build_system.py
│   └── submit.sh
│
├── notebooks/                 # 🟢 探索性 / 正式分析 Notebook
│   ├── 01_explore.ipynb
│   └── 02_rdf_analysis.ipynb
│
├── src/                       # 🟢 可选：给 import 用的模块
│   └── analysis/
│       └── rdf.py
│
├── data/                      # 🟠 模拟产出（大！）
│   ├── raw/                   # dump、log、lammpstrj —— 通常不进 Git
│   └── processed/             # csv、小表、裁剪轨迹
│
├── figures/                   # 🟣 论文图 PDF/PNG
│   └── fig1_rdf.pdf
│
└── docs/                      # 🟣 文字记录
    ├── runlog.md              # 每次 JOBID、参数、结论
    └── notes.md               # 组会、待办
```

**小课题：** 可不要 `src/`，逻辑全放 `scripts/`。**多温度、多体系：** 见第七节，把 `lammps/` 扩成 `simulations/case01/`。

## 三、各文件夹放啥、进不进 Git

| 目录 | 放什么 | 进 Git？ |
|------|--------|---------|
| `config/` | 力场、全局参数 | ✅ 要 |
| `structures/` | 初始 `data` / `xyz` / `pdb` | ✅ 一般不大 |
| `lammps/` | `in` 文件、`job.slurm` | ✅ 要 |
| `scripts/`、`notebooks/`、`src/` | 代码 | ✅ 要（Notebook 提交前清输出，见 [Jupyter Notebook科研使用规范](T16-Jupyter Notebook科研使用规范.md)） |
| `data/raw/` | 轨迹、大 log | ❌ 不要 |
| `data/processed/` | 精简 csv 等 | ⚠️ 单文件小于 10 MB 可进；再大就忽略 |
| `figures/` | 论文图 | ✅ 矢量 PDF 通常很小 |
| `docs/` | runlog、组会笔记 | ✅ 要 |

**一句话记：** 能 `git clone` 后靠脚本**再生成**的超大文件 → 别进仓库；**输入、代码、参数记录** → 必须进。

## 四、`.gitignore` 模板（抄走即用）

在项目根新建 `.gitignore`：

```gitignore
# Python
__pycache__/
*.pyc
.ipynb_checkpoints/
*.egg-info/

# 环境与密钥（别泄露集群账号）
.env
*.key

# 分子模拟大文件 —— 核心
data/raw/
*.dcd
*.xtc
*.trr
*.lammpstrj
*.dump
*.chk
*.restart

# 整个 data 都不要进 Git 时取消下一行注释
# data/

# 系统与编辑器垃圾
.DS_Store
.obsidian/workspace.json

# 临时编译
*.o
*.log.tmp
```

提交前 `git status` 扫一眼：若出现几百 MB 的 `.dcd`，说明 ignore 没写对或文件放错目录。详见 [Git简明使用教程](T04-Git简明使用教程.md)。

## 五、`README.md`：给「未来的你」看的说明书

根目录 `README.md` 建议至少四块：**项目是啥、环境怎么装、按什么顺序复现、大文件在哪**。

```markdown
# TIP3P 水盒子 NPT（示例项目）

## 环境
- `mamba env create -f environment.yml && conda activate md`
- 或 MolSimulX [分子模拟工作平台搭建](T01-分子模拟工作平台搭建.md) 里的 myenv

## 复现步骤
1. 初始结构：`structures/system.data`（Packmol + TIP3P）
2. 本地冒烟：`lmp -in lammps/in.minimize`（少步数）
3. 集群生产：`sbatch lammps/job.slurm`（见 [集群与SLURM简明教程](T10-集群与SLURM简明教程.md)）
4. 分析：打开 `notebooks/02_rdf_analysis.ipynb`

## 数据与图
- 原始轨迹：集群 scratch 或本地 `data/raw/`（未进 Git）
- 论文图：`figures/fig1_rdf.pdf`
- 每次运行记录：`docs/runlog.md`
```

导师、合作者、审稿人问「能复现吗」——先甩 README。

---

## 六、`docs/runlog.md`：每次模拟记一笔

比 README 更细：**每一次提交**记 JOBID、改了啥、结果咋样。模板：

```markdown
## 2026-06-10 run003 — NPT 生产

- **JOBID：** 123456（集群）
- **输入：** `lammps/in.npt`
- **力场 / 系综：** TIP3P，NPT 300 K，1 atm
- **步长 / 步数：** 1 fs，1e6 steps
- **随机种子：** 12345
- **输出：** `data/raw/run003/`（log + dcd）
- **结论：** 密度 0.99 g/cm³，与文献接近；下一步做 RDF
- **备注：** 第 2 次跑，修正了 kspace 精度
```

写论文 Methods、补充材料时，直接从这里抄参数表，不用翻聊天记录。

## 七、命名别踩坑

| 类型 | 建议 | 避免 |
|------|------|------|
| 文件夹 | 小写英文 + 下划线 `npt_300k` | 空格、中文路径（集群/WSL 易炸） |
| Lammps 输入 | `in.阶段`：`in.minimize`、`in.npt` | `新建文本文档.txt` |
| Notebook | 数字前缀 `01_explore`、`02_rdf` | 无顺序的 `test.ipynb` |
| 轨迹目录 | `water_npt_300k_20260610/` | 只有 `dump.lammpstrj` |
| 论文图 | `fig1_rdf.pdf` 与稿子里图号一致 | `未命名1.png` |

## 八、多案例 / 多体系怎么扩

一个仓库里跑多组温度、多个体系时：

```
my-md-project/
├── simulations/
│   ├── water_300k/
│   │   ├── lammps/
│   │   ├── data/          # 可 symlink 到大磁盘
│   │   └── notebooks/
│   └── water_350k/
│       └── ...
└── shared/
    ├── forcefield/
    └── scripts/           # 共用 rdf.py、submit 模板
```

每个子目录自己的 `job.slurm`；分析函数放 `shared/scripts/`，别复制粘贴十份。

## 九、和平台工具怎么配合

| 工具 | 怎么用这套目录 |
|------|----------------|
| **VSCode / Cursor** | `Open Folder` 打开项目根；Remote SSH 编辑集群上同一结构（见 [VSCode与Cursor远程连接集群](T07-VSCode与Cursor远程连接集群.md)） |
| **Git** | 根目录 `git init`；只跟踪代码、配置、`docs/` |
| **Obsidian** | `docs/` 当笔记夹，或整个项目当 Vault 子目录（见 [Obsidian知识库搭建](T13-Obsidian知识库搭建.md)） |
| **集群** | `rsync` 同步 `lammps/`、`scripts/`、`structures/`；轨迹留 scratch，定期拉回 `data/raw/`（见 [本地与集群文件传输](T09-本地与集群文件传输.md)） |
| **LaTeX** | `figures/` 里 PDF 直接 `\includegraphics`（见 [LaTeX与Overleaf简明教程](T14-LaTeX与Overleaf简明教程.md)） |

**典型一天：**

```
改 structures/ 或 lammps/in.npt
  → git commit
  → rsync 上集群 → sbatch lammps/job.slurm
  → 把 JOBID 记入 docs/runlog.md
  → 轨迹回 data/raw/ → notebooks/ 或 scripts/ 出图到 figures/
```

更完整的作图与投稿链路见 [从模拟到论文图的工作流](T18-从模拟到论文图的工作流.md)。

## 十、常见问题

### 1. `processed/` 里的小 csv 要不要进 Git？

能重新从轨迹算出来 → 可不进；算一次要几小时 → 可进 Git 或放网盘，在 README 里写清来源。

### 2. Notebook 要不要提交？

要，但**清掉大输出**再 commit，否则 `.ipynb` 也会胖成轨迹。见 [Jupyter Notebook科研使用规范](T16-Jupyter Notebook科研使用规范.md)。

### 3. 集群 scratch 被清空了怎么办

`runlog.md` 里写了 JOBID 和参数，输入还在 `lammps/` 里就能重跑；**不能只靠轨迹活命**。重要结果定期 `rsync` 到本地或组里 NAS（见 [数据管理与备份](T17-数据管理与备份.md)）。

### 4. 项目根该叫啥名

小写英文 + 连字符或下划线，如 `tip3p-water-npt`；别用空格。本地和集群**同一文件夹名**，`rsync` 最省心。

---

## 十一、小结

1. **项目根 = Git 仓库**；输入、代码、轨迹、图、日志分抽屉放。  
2. **轨迹进 `data/raw/`，写进 `.gitignore`**；参数进 `config/` 和 `docs/runlog.md`。  
3. **README** 写清复现四步；**runlog** 记每次 JOBID 与结论。  
4. 命名用英文 + 数字前缀；多案例用 `simulations/` + `shared/`。  
5. 与 Git、集群传文件、Notebook、出图教程串联，整条链路才走得通。
