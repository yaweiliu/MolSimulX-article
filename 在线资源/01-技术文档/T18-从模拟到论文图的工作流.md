> **系列标签：** `技术文档` · `工作流` · `论文` · `科研绘图`

论文里那一张 RDF、一条 MSD 曲线，背后往往不是「跑完顺手画一下」——而是从 **建结构 → 集群跑 Lammps → 读轨迹 → 算指标 → Matplotlib 出 PDF →（可选）版式与组合 → Overleaf 排版 → 补充材料** 一整条链。中间任何一环没记下来，Methods 写不清、图对不上数、审稿人一问复现就卡壳。

本文把 MolSimulX 里分散在多篇教程里的步骤**按时间串成一条线**，并给你分阶段检查清单。单点技能请跳转到对应专题；这里管「**先干啥、后干啥、交稿前对啥**」。

学完能带走：六阶段路线图（建模 → 跑模拟 → 分析 → 绘图 → 排版 → 投稿）和每步该落在哪个文件夹、交稿前该对哪几条——卡在某一步就跳文内链接，回来继续勾清单。目录规范见 [科研项目目录结构规范](T15-科研项目目录结构规范.md)；大文件和 runlog 见 [数据管理与备份](T17-数据管理与备份.md)；排版细节不在这里展开，见 [LaTeX与Overleaf简明教程](T14-LaTeX与Overleaf简明教程.md)。

## 二、阶段 1：建模与输入（Methods 的素材从这儿来）

| 步骤 | 放哪 / 用啥 | 产出 |
|------|-------------|------|
| 建结构 | ASE、Packmol、在线工具 | `structures/system.data` |
| 写输入 | VSCode / Cursor | `lammps/in.minimize`、`in.npt` |
| 记参数 | `docs/runlog.md` | 力场、系综、步长、种子 |
| 锁版本 | Git | 可复现的 commit |

**这一步的目标：** 别人拿到你的 Git 仓库，至少能知道**用的什么力场、什么系综、结构从哪来**。论文 Methods 第一段往往就抄这里的参数表 + 文献引用。

```bash
git add structures/ lammps/ docs/runlog.md
git commit -m "add: TIP3P 水盒子 NPT 输入"
```

> **Tips：** 本地先 **少步数** 跑通 `in.minimize`，再交长 NPT——别一上来 sbatch 一百万步，排队一天才发现 `pair_coeff` 写错。

## 三、阶段 2：跑模拟（算力在集群）

| 步骤 | 说明 |
|------|------|
| 冒烟测试 | 登录节点或交互节点小规模 `lmp -in in.minimize` |
| 正式生产 | `sbatch lammps/job.slurm`（见 [集群与SLURM简明教程](T10-集群与SLURM简明教程.md)） |
| 输出位置 | 轨迹写 **scratch** 或 `data/raw/runXXX/`，别塞满 `$HOME` |
| 立刻记 | runlog：**JOBID**、输出路径、提交时的 **Git commit** |

**跑完后先看 `log.lammps`：**

- 能量、温度、压强是否稳了？  
- 密度/盒子尺寸合理吗？  
- 不对就查 `in` 和力场，**别急着分析一条废轨迹**

模拟结束尽快 `rsync` 关键 run（见 [数据管理与备份](T17-数据管理与备份.md)、[本地与集群文件传输](T09-本地与集群文件传输.md)）。

## 四、阶段 3：轨迹分析（先试错，再定型）

| 步骤 | 工具 | 说明 |
|------|------|------|
| 探索 | `notebooks/01_*.ipynb` | 试读轨迹、看头几帧、粗算 RDF |
| 定稿 | `scripts/rdf.py` 等 | 路径固定，可 `python scripts/rdf.py` 重跑 |
| 读轨迹 | MDAnalysis | 见 [MDAnalysis轨迹分析入门](../01-技术文档/T22-MDAnalysis轨迹分析入门.md)；大文件注意别在登录节点硬扛 |
| 存数 | `data/processed/*.csv` | 画图只读 csv，改样式不用重读轨迹 |

规范见 [Jupyter Notebook科研使用规范](T16-Jupyter Notebook科研使用规范.md)：**探索用 Notebook，定稿进 scripts**；提交前 **Restart & Run All**。

**原则：** 脚本里读的轨迹路径 = runlog 里那条；换 run 只改一处配置（`params.yaml` 或脚本开头），别在十个 Notebook 里各写死路径。

## 五、阶段 4：科研绘图（Matplotlib → 论文图）

在 [NumPy与Matplotlib简明教程](T21-NumPy与Matplotlib简明教程.md) 基础上，**投稿图**多记四件事：

### 1. 尺寸：按期刊栏宽来

```python
# 单栏约 3.5 in 宽，双栏约 7 in（以目标期刊 Author Guide 为准）
fig, ax = plt.subplots(figsize=(3.5, 2.8))
ax.plot(r, g_r)
ax.set_xlabel(r"$r$ (Å)")
ax.set_ylabel(r"$g(r)$")
fig.savefig("figures/fig2_rdf.pdf", bbox_inches="tight")   # 投稿主用
fig.savefig("figures/fig2_rdf.png", dpi=300, bbox_inches="tight")  # 预览
```

### 2. 单位与符号

- 轴标签带单位；Lammps `units real` 下长度常用 Å，时间 fs  
- 公式用 `$g(r)$`、`$D$`，和 [LaTeX与Overleaf简明教程](T14-LaTeX与Overleaf简明教程.md) 一致  

### 3. 色盲友好配色

读者里相当一部分人有**色觉异常**（俗称「色盲」），其中 **红–绿分不清**最常见。若图里只有「红线 = 300 K、绿线 = 350 K」，印成黑白或对方屏幕偏色时，两条线可能**看起来几乎一样**——审稿人、答辩委员都可能误读。

**实用做法（可叠加使用）：**

| 做法 | 说明 |
|------|------|
| **换色板** | 用专为色盲筛选过的配色，别只靠红/绿一对 |
| **线型 + 标记** | 实线/虚线配合 `o`、`s` 等不同 marker |
| **图例写清** | 颜色不够用时，靠文字标签区分条件 |

**推荐资源（选图前看一眼）：**

- [ColorBrewer](https://colorbrewer2.org/) — 勾选 **colorblind safe** 选定性/顺序色板  
- [Paul Tol 配色笔记](https://packages.tesselle.org/khroma/articles/tol.html) — 科研图常用，提供 PDF 色表与说明  
- [Matplotlib 色盲主题 `tableau-colorblind10`](https://matplotlib.org/stable/gallery/style_sheets/style_sheets_reference.html) — 内置样式，开箱可用  
- [Coblis 色盲模拟器](https://www.color-blindness.com/coblis-color-blindness-simulator/) — 上传成图，预览色盲读者看到的效果  

Matplotlib 示例：

```python
import matplotlib.pyplot as plt

# 方式 A：整图用色盲友好样式
plt.style.use("tableau-colorblind10")

# 方式 B：手动指定 Tol 系常用色（多曲线）
colors = ["#0173B2", "#DE8F05", "#029E73", "#CC78BC"]
for c, T in zip(colors, [300, 320, 340, 360]):
    ax.plot(r, g_dict[T], color=c, linestyle="-", marker="o", markevery=20,
            label=f"{T} K")
```

更多绘图细节见 [NumPy与Matplotlib简明教程](T21-NumPy与Matplotlib简明教程.md) 中的 ColorBrewer 说明。

### 4. 多 panel（a）（b）

```python
fig, axes = plt.subplots(1, 2, figsize=(7, 3), sharex=True)
for ax, T in zip(axes, [300, 350]):
    ax.plot(r, g_dict[T], label=f"{T} K")
    ax.legend()
axes[0].set_ylabel(r"$g(r)$")
fig.savefig("figures/fig3_combined.pdf", bbox_inches="tight")
```

### 5. 误差与统计

多个独立初始构型或时间分块平均时，加误差棒；**样本数、块长**写进 caption 或 Methods。

**图文件命名：** `figures/fig2_rdf.pdf` 与稿子里 Figure 2 对应；runlog 里写「fig2 ← run030」。

## 六、阶段 5：精修（可选）

Matplotlib 直接出的 PDF **很多期刊已够用**。要统一线宽、加箭头、拼结构图时：

| 工具 | 干啥 |
|------|------|
| **Inkscape** | 免费，改 PDF 矢量 |
| **Illustrator** | 同上（付费） |
| **VMD / OVITO** | 分子结构渲染 → 导出 SVG/PDF 再和曲线拼版 |

流程：**Matplotlib PDF → Inkscape 微调 → 仍导出 PDF → Overleaf `\includegraphics`**。别最后只剩 72 dpi 的截图。

## 七、阶段 6：进 LaTeX / 投稿

```latex
\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.48\textwidth]{figures/fig2_rdf.pdf}
  \caption{TIP3P 水在 300~K、1~atm 下的径向分布函数.
           轨迹长 10~ns，系综 NPT。}
  \label{fig:rdf}
\end{figure}
```

详见 [LaTeX与Overleaf简明教程](T14-LaTeX与Overleaf简明教程.md)。

**图文一致三件套：**

1. 图上的曲线 = `data/processed/` 里那份数据  
2. caption 参数 = `docs/runlog.md` 对应条目  
3. Methods 力场/步长 = Git 里那次 commit 的 `lammps/` 与 `config/`

## 九、端到端检查清单

**模拟前**

- [ ] [科研项目目录结构规范](T15-科研项目目录结构规范.md) 目录已建好  
- [ ] `in`、结构已 `git commit`  
- [ ] runlog 模板在 `docs/` 里  

**模拟后**

- [ ] `log.lammps` 看过，能量/温压合理  
- [ ] runlog 有 JOBID、scratch 路径、commit  
- [ ] 关键 raw 已 `rsync` 出 scratch  

**分析后**

- [ ] Notebook **Run All** 通过，或 `python scripts/xxx.py` 可重跑  
- [ ] `data/processed/` 与图一致  

**投稿前**

- [ ] 图为 **PDF 矢量**，轴标签带单位  
- [ ] Overleaf 里 `\ref{fig:...}`、caption 无误  
- [ ] Methods 能根据 Git + runlog **复现**  
- [ ] 补充材料 / 数据 DOI（若期刊要求）已准备

## 十、常见问题

### 1. 图很好看，但和 Methods 对不上

多半是 runlog 没更新，或 Notebook 里手改温度没重跑。**以 runlog + Git commit 为权威**，重画前先对路径。

### 2. 只有 Notebook 没有脚本

短期可以；要投稿、要合作，把稳定流程迁到 `scripts/`（见 [Jupyter Notebook科研使用规范](T16-Jupyter Notebook科研使用规范.md)）。

### 3. 结构图是截图，曲线是 PDF

审稿放大糊掉。结构也用矢量导出，或在 Inkscape 里和曲线同一画布。

### 4. 整条链太长，从哪练？

跟一篇 [NumPy与Matplotlib简明教程](T21-NumPy与Matplotlib简明教程.md) **用假数据练手**，先走通 4→5→6，再接真实 Lammps 轨迹；完整案例见 `02-实战案例`（陆续更新）。

## 十一、小结

1. **输入 + runlog + Git** 决定 Methods 能不能写清楚。  
2. **分析**：Notebook 探索 → `scripts/` + `processed` 定稿。  
3. **图**：Matplotlib 出 **PDF** → 可选 Inkscape → **Overleaf**。  
4. 目录与备份贯穿全程（[科研项目目录结构规范](T15-科研项目目录结构规范.md)、[数据管理与备份](T17-数据管理与备份.md)）；别等投稿前才找轨迹。
