脚本在笔记本上写，Lammps 在集群上跑，轨迹和分析结果还得拉回本地——**本地和集群之间倒文件**是分子模拟里的家常便饭。传错目录、传到一半断了、或者把几 GB 轨迹塞进 Git，都能折腾半天。

本文讲几种常用办法：`scp`、`rsync`、终端 `sftp`，还有 **FileZilla** 这类图形工具，以及 VSCode Remote SSH 怎么配合。传之前先把 SSH 配好，见 [SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md)；登录节点、scratch 等概念见 [集群与SLURM简明教程](T10-集群与SLURM简明教程.md)。

学完能带走：小文件偶尔传用 `scp`、项目增量同步用 `rsync`、不想敲命令用 FileZilla，以及代码和几 GB 轨迹该丢家目录（HOME）还是大容量盘（SCRATCH）。连不上集群先翻 [SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md)；改 in 文件、少来回 scp 可配合 [VSCode与Cursor远程连接集群](T07-VSCode与Cursor远程连接集群.md)；传完怎么归档见 [数据管理与备份](T17-数据管理与备份.md)。

## 一、传之前：先想清楚三件事

| 问题 | 含义 |
|------|------|
| **传什么？** | 小的：`lammps/`、`scripts/`、结构文件；大的：dump、xtc、dcd 轨迹 |
| **传到哪？** | 家目录（HOME）像「宿舍柜子」——放代码、脚本；大容量盘（SCRATCH / WORK）像「临时大仓库」——放轨迹，可能定期清空 |
| **在哪传？** | 一般在**登录节点**收发文件；传文件没问题，**别在这儿跑 Lammps** |

> **Tips：** 各集群 HOME、SCRATCH 叫什么、配额多少、校外能不能直传，**都不一样**——看本校手册或问管理员，别抄别校的目录名。

目录规划见 [科研项目目录结构规范](T15-科研项目目录结构规范.md)；传完后怎么归档见 [数据管理与备份](T17-数据管理与备份.md)。

## 二、先把 SSH 配好（一次就行）

`scp`、`rsync`、`sftp`、FileZilla 都走 SSH。密钥、`~/.ssh/config` 别名、权限怎么弄，完整步骤见 **[SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md)**。这里只确认：**终端里 `ssh molcluster` 能上去**再传文件。

### 1. 速览：`config` 与测试

```
Host molcluster
    HostName cluster.university.edu.cn
    User your_username
    IdentityFile ~/.ssh/id_ed25519
    ServerAliveInterval 60
```

```bash
ssh molcluster    # 测试免密登录
```

### 2. 集群侧路径约定（示例）

```bash
ssh molcluster
echo $HOME
echo $SCRATCH    # 或 $WORK、/scratch/$USER，因集群而异
mkdir -p ~/project ~/scratch/project/raw
```

**记一句：** 代码放 `~/project`；轨迹输出指到 scratch。具体路径以集群手册为准。

## 三、`scp`：简单粗暴的复制

`scp` 就是「安全版的 cp」，适合**偶尔传几个小文件**。记格式：

```bash
scp [选项] 源 目标
```

远程路径写成 `molcluster:路径`（别名来自 [SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md) 第五节）。

### 1. 本地上传 → 集群

```bash
# 单个文件
scp in.npt molcluster:~/project/lammps/

# 整个目录（-r 递归）
scp -r lammps/ scripts/ molcluster:~/project/

# 从项目根目录一次传多个
scp -r structures/ lammps/ job.slurm molcluster:~/project/
```

### 2. 集群下载 → 本地

```bash
# 拉回 log
scp molcluster:~/project/log.lammps ./data/raw/

# 拉回整个 processed 目录
scp -r molcluster:~/project/data/processed/ ./data/processed/
```

### 3. 常用选项

| 选项 | 含义 |
|------|------|
| `-r` | 整个目录递归传 |
| `-C` | 路上压缩（网速慢时可试） |
| `-p` | 保留修改时间 |
| `-l 50000` | 限速，别占满带宽 |

```bash
scp -C -r figures/ molcluster:~/project/figures/
```

### 4. 注意

- 几 GB 轨迹**能**用 `scp`，但断了要重头来；日常更推荐 `rsync`  
- 路径有空格加引号：`scp "local my file.dat" molcluster:~/project/`  
- **方向别反**：`本地 远程` = 上传；`远程 本地` = 下载

## 四、`rsync`：日常首选

`rsync` 只传**有变化**的部分，还能 `--partial` 断点续传——传轨迹、`data/` 目录都靠它。

基本格式：

```bash
rsync [选项] 源/ 目标/
```

> **Tips：** 源路径末尾的 `/` 很要紧：  
> `dir/` = 传目录**里面**的东西；`dir` = 连目录名一起传过去。  
> 拿不准就先加 `-n` **试跑**，看会动哪些文件。

### 1. 常用选项组合

```bash
rsync -avh --progress 源/ 目标/
```

| 选项 | 含义 |
|------|------|
| `-a` | 归档模式（权限、时间、递归，相当于 -rlptgoD） |
| `-v` | 显示详情 |
| `-h` | 人类可读的文件大小 |
| `--progress` | 进度条 |
| `--partial` | 保留部分传输的文件，便于续传 |
| `-n` / `--dry-run` | **试跑**：只显示会传什么，不真传 |
| `--delete` | 目标侧删除源已不存在的文件（**慎用**，会删远程多余文件） |

### 2. 上传项目代码（排除大文件）

```bash
cd ~/projects/my-md-project

# 先试跑
rsync -avhn --progress \
  --exclude 'data/raw/' \
  --exclude '.git/' \
  --exclude '__pycache__/' \
  --exclude '.ipynb_checkpoints/' \
  ./ molcluster:~/project/

# 确认无误后去掉 -n
rsync -avh --progress \
  --exclude 'data/raw/' \
  --exclude '.git/' \
  ./ molcluster:~/project/
```

与 [科研项目目录结构规范](T15-科研项目目录结构规范.md) 中的 `.gitignore` 思路一致。

### 3. 下载分析结果

```bash
# 只拉回 processed 与 figures
rsync -avh --progress \
  molcluster:~/project/data/processed/ ./data/processed/

rsync -avh --progress \
  molcluster:~/project/figures/ ./figures/
```

### 4. 大轨迹（scratch ↔ 本地）

```bash
# 从集群 scratch 拉回（路径按实际修改）
rsync -avh --progress --partial \
  molcluster:/scratch/$USER/project/run030/ ./data/raw/run030/
```

上传大轨迹到集群（较少见，通常直接在集群生成）：

```bash
rsync -avh --progress --partial \
  ./data/raw/run030/ molcluster:/scratch/$USER/project/run030/
```

### 5. 保持两边同步（进阶）

本地改代码后反复上传：

```bash
rsync -avh --progress --delete \
  --exclude 'data/' \
  ./ molcluster:~/project/
```

`--delete` 会使远程与本地严格一致；确认不会误删远程独有文件再用。

## 五、`sftp`：终端里逛远程目录

路径不熟、想先看看集群上有什么，可以进 `sftp` 交互模式：

```bash
sftp molcluster
```

| 你想做的事 | 命令 |
|-----------|------|
| 看远程文件 | `ls` / `ls -l` |
| 看本地文件 | `lls` |
| 切远程目录 | `cd 路径` |
| 切本地目录 | `lcd 路径` |
| 下载 | `get 远程文件 [本地路径]` |
| 上传 | `put 本地文件 [远程路径]` |
| 批量下载 | `mget *.log` |
| 断点续传下载 | `reget 大文件.xtc` |
| 退出 | `bye` |

图形化传小文件可用 Remote SSH 资源管理器或 **FileZilla**（见下节）；**大轨迹**仍用终端 `rsync`。远程连接完整配置见 [VSCode与Cursor远程连接集群](T07-VSCode与Cursor远程连接集群.md)。

## 六、图形化：FileZilla 等（可选）

命令行不熟也没关系——**SFTP 图形客户端**左右两栏拖文件，和 `sftp` 走同一条 SSH 隧道。前提是 [SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md) 密钥已经配好。

### 1. 和命令行怎么分工？

| 方式 | 更适合 |
|------|--------|
| **FileZilla / WinSCP** | 逛目录、拖几个 `in` / 脚本 / 结构文件 |
| **`rsync`** | 数 GB 轨迹、增量同步、断点续传 |
| **`scp`** | 临时传一两个小文件 |
| **Remote SSH** | 直接在集群上改代码，省得来回传 |

**一句话：** 图形工具省心，**大轨迹别硬拖**——慢、易断、难续传；轨迹一律 `rsync --partial`。

### 2. FileZilla（Mac / Windows / Linux）

[FileZilla Client](https://filezilla-project.org/) 免费，协议选 **SFTP**（不是 FTP）。

**站点管理器里新建站点：**

| 字段 | 填什么 |
|------|--------|
| 协议 | **SFTP - SSH File Transfer Protocol** |
| 主机 | `config` 里的 `HostName`（或完整域名，不能只写 `molcluster` 别名） |
| 端口 | 22（或集群规定的端口） |
| 登录类型 | **密钥文件** |
| 用户 | 集群用户名 |
| 密钥文件 | 本机私钥，如 `~/.ssh/id_ed25519`（WSL 内路径） |

> FileZilla **不读** `~/.ssh/config` 里的 `Host` 别名；别名里的 `HostName`、`User`、`IdentityFile` 要**手工填进站点**。跳板机 `ProxyJump` 场景图形客户端往往不好配，仍建议终端 `ssh` / `rsync` 或问管理员是否有 **数据传输节点**。

连接成功后：左侧本地、右侧远程，拖文件即可。右键可建目录、改权限（视集群策略而定）。

### 3. 其他常见客户端

| 软件 | 平台 | 含义 |
|------|------|------|
| **WinSCP** | Windows | 和 FileZilla 类似，很多人用惯 |
| **Cyberduck** | Mac / Windows | 轻量 SFTP，偶尔传文件够用 |
| **MobaXterm** | Windows | SSH + SFTP 一体，功能多但略重 |
| **VS Code / Cursor** | 全平台 | 侧边栏拖小文件，见第七节 |

部分学校超算还提供 **Globus** 网页传大数据——有的话优先问管理员，往往比 GUI 拖 GB 文件更稳。

### 4. 使用注意

- 协议务必 **SFTP**，不要用明文 FTP  
- 私钥路径选对（WSL 用户用 `/home/你/.ssh/...`，不是 `C:\...`）  
- 传到 scratch 大目录前，确认路径和配额（第一节）  
- 图形工具传的脚本，路径仍建议和 [科研项目目录结构规范](T15-科研项目目录结构规范.md) 一致

## 七、VSCode / Cursor Remote SSH

装好 **Remote - SSH** 后，在集群上直接打开 `~/project` 改文件——**保存就是写到集群**，改几个 `in` 或脚本时往往不必 `scp`。

| 场景 | 怎么做 |
|------|--------|
| 改输入文件、Python 脚本 | Remote SSH 打开项目文件夹 |
| 传几 GB 轨迹 | 还是得终端 `rsync`（编辑器不适合搬大文件） |
| 拖个小文件 | 资源管理器拖拽可以；大的别拖 |

详见 [VSCode与Cursor远程连接集群](T07-VSCode与Cursor远程连接集群.md)。

**别误会：** Remote SSH **不会**自动把本地 `data/raw/` 同步上去；大文件照样走 `rsync`。

---

## 八、Windows 用户注意

| 环境 | 建议 |
|------|------|
| **WSL** | 在 WSL 终端执行 `scp`/`rsync`；项目在 `~/projects` |
| **从 Windows 盘传** | 路径 `/mnt/c/Users/你/...`，较慢；大文件先拷进 WSL 家目录 |
| **Git Bash** | 可用 `scp`，一般无 `rsync`；可 `sudo apt install rsync`（在 WSL 里）或装 WSL |
| **FileZilla / WinSCP** | Windows 上图形传小文件；密钥选 WSL 内 `\\wsl$\Ubuntu\home\你\.ssh\id_ed25519` 或 Windows 侧 `.ssh`（与本文 WSL 路线一致） |
| **PuTTY / psftp** | 老环境备选，需单独转换密钥格式（`.ppk`） |

项目与 Conda 建议放在 WSL 内，与集群 Linux 环境一致（[WSL2安装与配置](T02-WSL2安装与配置.md)）。

## 九、分子模拟场景速查

| 任务 | 推荐命令 / 方式 |
|------|-----------------|
| 首次部署项目到集群 | `rsync` 上传，排除 `data/raw/` |
| 改了几个 `in` / 脚本 | Remote SSH 保存，或 `rsync` 增量 |
| 浏览远程目录、拖几个小文件 | FileZilla / WinSCP（SFTP） |
| 模拟完成，拉回 log | `scp molcluster:~/project/log.lammps ./data/raw/` |
| 拉回整条轨迹 | `rsync -avh --progress --partial` 从 scratch |
| 只拉回分析用 csv | `rsync` `data/processed/` |
| 备份到本地硬盘 | `rsync` 到 `/Volumes/Backup/...` |
| 组内共享 | 集群共享目录 + 权限；或网盘（见数据管理教程） |

**别干这两件事：** 把 `*.dcd` / `*.xtc` 塞进 Git 当传输方式；在登录节点一边传几小时大文件一边跑 Lammps。

## 十、传得快、传得稳

### 1. 校园网 / VPN

校外访问集群常需 **VPN**；传大文件前确认连接稳定。

### 2. 压缩

文本类（log、in、csv）可加 `-z`（rsync）或 `scp -C`。**已压缩的轨迹**（xtc）再压缩收益小。

### 3. 中断后续传

```bash
rsync -avh --progress --partial --append-verify \
  molcluster:/scratch/user/run030/dump.lammpstrj ./data/raw/
```

### 4. 并行（多文件，进阶）

```bash
# GNU parallel 或手动开多个 rsync 传不同子目录
rsync -avh run030/ molcluster:/scratch/user/run030/ &
rsync -avh run031/ molcluster:/scratch/user/run031/ &
wait
```

### 5. 校验（可选）

```bash
# 比较本地与远程是否一致（不传输，只对比）
rsync -avnc molcluster:~/project/ ./local-mirror/
```

或传完后：`md5sum file` 与远程 `ssh molcluster md5sum path` 对比。

## 十一、安全与规范

- 勿在 `scp`/`rsync` 命令行写密码；用 SSH 密钥  
- 敏感未发表数据：加密磁盘、权限正确的目录，勿传公共网盘无密码链接  
- 遵守集群**出口流量与存储配额**；scratch 满时先归档再传  
- 用 `rsync -n` 先试跑，尤其带 `--delete` 时  

---

## 十二、常见问题

### 1. `Permission denied (publickey)`

多半是 SSH 没配好：查 `~/.ssh/config`、`IdentityFile` 权限（`chmod 600`）、公钥是否上了集群。详见 [SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md)；`ssh -v molcluster` 看详细日志。

### 2. `No such file or directory`（远程路径）

先 `ssh molcluster`，`mkdir -p` 建好目录；SCRATCH 有的集群要登录后才有环境变量，先 `echo $SCRATCH` 看一眼。

### 3. 速度极慢

高峰时段正常；换 VPN 线路、文本文件试 `-C` / `-z`；问管理员有没有**数据传输节点**或 **Globus**。

### 4. `rsync: command not found`（Windows）

在 **WSL** 里装：`sudo apt install rsync`；或暂时用 `scp -r`。

### 5. 传完了脚本却找不到文件

本地和集群**目录结构保持一致**，脚本里用相对路径；见 [科研项目目录结构规范](T15-科研项目目录结构规范.md)。

## 十三、命令速查

| 你想做的事 | 命令 / 工具 |
|-----------|-------------|
| 上传文件 | `scp file molcluster:~/project/` |
| 上传目录 | `scp -r dir/ molcluster:~/project/` |
| 下载文件 | `scp molcluster:~/project/log.lammps ./` |
| 增量同步上传 | `rsync -avh --progress ./ molcluster:~/project/` |
| 增量同步下载 | `rsync -avh --progress molcluster:~/project/data/ ./data/` |
| 试跑不同步 | `rsync -avhn ...` |
| 交互浏览（终端） | `sftp molcluster` |
| 图形拖拽（小文件） | FileZilla / WinSCP，协议 **SFTP** |

---

## 十四、小结

1. **三板斧**：小文件 `scp`；日常和轨迹 `rsync -avh --progress`；逛目录 FileZilla / `sftp`。  
2. **大轨迹**务必 `rsync --partial`；图形工具只拖小的。  
3. SSH 别名见 [SSH密钥与config配置简明教程](T08-SSH密钥与config配置简明教程.md)；FileZilla 要手工填 `HostName` 和私钥。  
4. 代码进 `~/project`，轨迹进 **scratch**；上传时排除 `data/raw/`。  
5. **改代码**用 Remote SSH；**搬轨迹**用 `rsync`——别混用。  
6. 路径和配额**问管理员**；传不动时把 `rsync` / `ssh -v` 的完整输出复制去搜。
