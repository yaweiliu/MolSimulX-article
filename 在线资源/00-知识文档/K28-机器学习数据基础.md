## 一、机器学习在说什么？

用数据 $\{(x_i, y_i)\}$ 拟合映射 $f$，使 $f(x)$ 能预测新样本的标签或结构：

| 类型 | 标签 $y$ | MD 相关例子 |
|------|------------|-------------|
| **回归** | 连续数值 | 扩散系数、形成能、沸点 |
| **分类** | 离散类别 | 相态、是否折叠、晶体/非晶 |
| **聚类** | 无标签 | 构象分群、轨迹态发现 |
| **降维** | 无监督 | 高维坐标 → 2D 可视化 |

---

## 二、特征（X）与标签（y）

### 1. 特征

描述每个**样本**的数值向量或结构对象：

| 来源 | 特征示例 |
|------|----------|
| 结构 | 组分、密度、键长统计、RDF 峰 |
| 轨迹 | RMSD 序列统计、序参量时间平均 |
| 分子 | RDKit 描述符、指纹 |
| 量子 | 轨道能量、电荷（作输入或标签） |

**一行 = 一个样本**；列 = 特征维度。多个特征量纲差大时常需**标准化**（技术示例中 `StandardScaler`）。

### 2. 标签

| 来源 | 说明 |
|------|------|
| 实验 | 溶解度、活性、熔点 |
| DFT / QM | 形成能、反应能垒 |
| 模拟 | 用更高精度势或更长 MD 作「伪真值」 |
| 规则 | 由序参量阈值定义相态（需谨慎） |

标签噪声与实验条件须在 Methods 中说明。

## 三、训练集、验证集、测试集

| 集合 | 用途 |
|------|------|
| **训练** | 拟合模型参数 |
| **验证** | 调超参、早停、选模型 |
| **测试** | **最终一次**评估泛化，不参与任何调参 |

简单划分：`train_test_split`；更稳：**k 折交叉验证**（见 sklearn 教程）。

**分子模拟特有划分原则：**

| 做法 | 原因 |
|------|------|
| 按**分子 / 独立模拟**划分 | 同一轨迹相邻帧高度相关 |
| 按**化学空间**划分 | 测试集含未见元素/骨架 |
| 勿把同一体系不同帧拆到 train/test | **信息泄漏**，指标虚高 |

---

## 四、信息泄漏（常见坑）

| 泄漏类型 | 例子 |
|----------|------|
| 帧级泄漏 | 第 100–200 帧训练，第 201–300 帧测试（同一轨迹） |
| 预处理泄漏 | 用**全数据**算均值方差再划分 |
| 标签泄漏 | 标签由特征直接算出再预测 |
| 重复样本 | 数据增强后同分子多行未去重 |

正确做法：先划分，再在训练集上 `fit` 缩放器；Pipeline 可自动化（sklearn 教程）。

## 五、数据量与质量

| 问题 | 影响 |
|------|------|
| 样本太少 | 过拟合，测试不稳 |
| 标签不准 | 上限低，模型学噪声 |
| 特征无信息 | 再复杂模型无用 |
| 分布偏移 | 训练液体、测试固体则失效 |

**MD 生成数据**时记录：力场、T、P、步长、随机种子（[数据管理与备份](../01-技术文档/T17-数据管理与备份.md)）。

## 六、单位与量纲

| 要求 | 说明 |
|------|------|
| 统一单位 | Å vs nm、kcal/mol vs eV |
| 记录 metadata | 每列特征含义与单位 |
| 无量纲化 | 有时用约化单位 |

论文图与 ML 特征必须一致，否则模型与物理对不上。

## 八、经典 ML vs 深度学习数据形态

| | 经典 ML（sklearn） | 深度学习（PyTorch） |
|--|-------------------|---------------------|
| 输入 | 二维表 `(N, F)` | Tensor，可更高维 |
| 结构数据 | 描述符 | 图、点云、网格 |
| 样本量 | 几十–几万常够用 | 大数据更吃香 |

神经网络见 [神经网络与深度学习基础](K29-神经网络与深度学习基础.md)。

## 十、小结

1. 分清 **特征 X** 与 **标签 y**；一行一样本。  
2. **按独立模拟/分子划分**，避免轨迹帧泄漏。  
3. 缩放、调参只用训练（或训练+验证），测试集留到最后。  
4. **单位与 metadata** 与 MD 工作流一并管理。
