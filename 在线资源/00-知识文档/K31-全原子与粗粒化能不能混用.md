短答先放桌上：**多数时候，不太行。**  
不是「软件报不了错」，而是把两套**各自成套标定**的参数硬拼在一起，盒子里的有效相互作用往往既不是全原子（AA），也不是粗粒（CG）——等于另造了一个没人验证过的力场。

本篇把「看起来能混」和「其实在换模型」分开说。力场怎么选见 [力场怎么选](K06-力场怎么选.md)；CG 梯子见 [粗粒化与加速模型](K04-粗粒化与加速模型.md)；同分辨率里 AA 族之间也不能乱拆，见 [经典全原子力场](K03-经典全原子力场.md)。

---

## 一、默认立场：不要 DIY 混分辨率

| 你想做的事 | 更稳妥的默认 |
|------------|--------------|
| 蛋白用 AMBER，膜用 Martini 珠，水用 TIP3P | **别这么干**（除非用专门的混合力场 / 多尺度方案） |
| 溶质全原子，溶剂换成 mW / Martini 珠水「省钱」 | **别默认能混**；等于换溶剂环境 |
| UA 烷烃参数 + 全原子水/电荷规则随手拼 | **别拼**；UA 与 AA 各有一套类型与排除 |
| 先 CG 粗扫，再 backmap 到 AA 细校 | **可以**——这是**先后**用两套模型，不是同一哈密顿里混粒子 |

经验上：能跑出轨迹 ≠ 热力学 / 界面张力 / 分配行为靠谱。静默偏差很常见——能量看着平滑，结构却系统性偏。

> **Tips：** 审稿人若问「溶质 AA、溶剂 CG，参数如何一致？」，答不上「成套文献 + 验证过的跨分辨率项」，就说明混用还没立住。

---

## 二、为什么硬混往往不行

### 1. 参数是「成套有效」的，不是乐高积木

AA 的 LJ、电荷、键参数，是在**某一分辨率、某套水、某套排除 / 1–4**下对着实验或更高精度拟合的。CG 珠参数同样是在**珠–珠、珠–珠水**的世界里拟合有效排斥、亲疏水、有时还有熵贡献（见 [粗粒化动力学加速与耗散](K30-粗粒化动力学加速与耗散.md)）。

把 AA 原子和 CG 珠放进同一非键表时，你必须凭空发明：

- 原子–珠之间的 $\varepsilon$、$\sigma$（或软势强度）；  
- 电荷怎么过界面、介电常数用谁的；  
- 排除与近邻列表按哪套拓扑。  

缺了专门标定，这三项通常**没有唯一正确答案**——软件用 Lorentz–Berthelot 之类默认混合规则「凑」出来的交叉项，并不保证物理。

### 2. 长度、能量、时间尺子本来就不齐

| 维度 | 硬混时容易出的事 |
|------|------------------|
| **长度** | 珠的有效直径与原子 LJ $\sigma$ 不对齐 → 界面密度、渗透、堆积全歪 |
| **能量** | 珠的有效势已含平均 / 熵，原子势更「真空」→ 吸附、分配系数不可比 |
| **时间** | CG 侧动力学常偏快，AA 侧更慢 → 同一盒子里两侧「时钟」不一致（流变 / 扩散更惨） |

### 3. 溶剂是半壁江山

最常见翻车：**全原子溶质 + 粗粒水**（或反过来）。水模型与溶质参数在 AA 里本就要求成套（TIP3P 配某族、SPC/E 配另一套习惯……）。换成珠水 / mW 后：

- 氢键几何、介电、表面张力都变了；  
- 溶质的水化壳、疏水效应、盐桥强度跟着变；  
- 你省下的溶剂粒子，往往以「错误溶剂环境」买单。

Martini 标准水本身就是粗粒 LJ 型珠水，须与 Martini 溶质珠成套——不是 TIP 水的换皮（见 [粗粒化与加速模型](K04-粗粒化与加速模型.md) 粗粒水一节）。

## 三、哪些「像混用」，其实不是一回事

| 做法 | 算不算本篇说的混用？ | 说明 |
|------|----------------------|------|
| **同一 AA 族内**换配套水（文档允许的那种） | 否 | 仍是全原子成套问题，见 [力场怎么选](K06-力场怎么选.md) |
| **先后**：CG 跑通 → backmap → AA 生产 | 否 | 两阶段、两套哈密顿；交接要写清 |
| **文献里的混合 / 多尺度力场**（专为 AA–CG 界面参数化） | 特例 | **可以**，但必须整套采用，不能只借一半 |
| **自适应分辨率**（如部分区域 AA、部分 CG，带过渡力） | 特例 | 方法本身就是课题；有额外自由度与伪力，入门勿当默认 |
| **自己把 AA 拓扑和 Martini `.itp` 拼进一个盒子** | **是，且高危** | 本篇默认反对的那种 |

口诀：**成套的混合方案 ≠ 随手混粒子类型。**

## 四、若课题真需要「细的一块 + 粗的一片」

按成本从低到高想：

1. **先问能不能全程同一分辨率**  
   - 只要介观形态 / 相分离 → 全程 CG；  
   - 只要局部化学 / 氢键 → 缩小盒子，全程 AA（或增强采样）。  

2. **先后耦合（推荐默认）**  
   粗模型找候选构型 / 相区 → 映射回 AA → 在 AA 上平衡与计算关键自由能。不在同一非键列表里发明交叉项。  

3. **选用已发表的混合力场或官方多尺度流程**  
   读清：哪些粒子用 AA、哪些用 CG、交叉项如何定义、水用哪套、有哪些已知偏差。把文献当 Methods 的一部分，而不是只抄珠类型名。  

4. **自己发明交叉参数**  
   等于新力场工程：要对结构、分配、界面张力、最好还有相关实验或 AA 参照做验证。工作量常超过「老老实实选一边」。

> **Tips：** 写进论文的「混合分辨率」必须能回答三句：交叉相互作用怎么来的？和谁的实验 / AA 对过？局限是什么？答不全，审稿风险很高。

## 五、实践检查清单

| 检查 | 过关标准 |
|------|----------|
| 分辨率 | 同一生产段里，是否只有**一套**文档承认的分辨率组合？ |
| 水 / 溶剂 | 与溶质、膜、离子是否同一力场族推荐？ |
| 交叉项 | 若真有 AA–CG 接触：参数出处是文献成套，还是软件默认混合？ |
| 时间尺度 | 是否在解释扩散 / 流变时假装两侧时钟相同？（见 [粗粒化动力学加速与耗散](K30-粗粒化动力学加速与耗散.md)） |
| 替代方案 | 能否改成「先后 CG→AA」或「缩小体系全程 AA」而少掉交叉项？ |

## 六、小结

1. **默认：全原子与粗粒化不要 DIY 混在同一哈密顿里**——参数成套、尺子不齐、溶剂尤其敏感。  
2. 能出轨迹不等于物理对；交叉 LJ / 电荷用默认混合规则，常常是静默错误。  
3. **先后多尺度**（粗扫再细校、backmapping）和**文献成套混合力场**，与「随手拼粒子」不是一类事。  
4. 真要细+粗同框，优先找已验证方案；自己拟合交叉项等于新力场，验证成本高。  
5. 选型仍回 [力场怎么选](K06-力场怎么选.md)：先问问题需要哪层分辨率，再决定要不要碰混合。
