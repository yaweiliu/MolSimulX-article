> **系列标签：** `实战案例` · `受限流体` · `界面建模` · `SPC/E` · `MDStudio`

想模拟「水在两块固体板之间」——电极界面的双电层、纳米孔道里的受限水、云母/石墨表面的润湿层——第一步就会卡住：普通体相模拟只要把分子丢进一个周期盒子跑 NPT 就行，可界面体系不一样。你要的是一个**板-水-板三明治**：上下两片固体基质把水夹在中间，板基本不动、只提供界面，水在缝隙里自己排布。直接往盒子里乱塞分子，水会钻进板里、或者把板挤变形。

这篇用 **MDStudio** 把这个三明治搭出来：结构与力场用[力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md)（或[周期分子](../../在线工具/00-MDStudio/M07-MDStudio周期分子.md)、[分子仓库](../../在线工具/00-MDStudio/M08-MDStudio分子仓库.md)）备齐，装盒用[搭建盒子](../../在线工具/00-MDStudio/M11-MDStudio搭建盒子.md)。整条流水线里**唯一需要手动介入的地方**，是把搭建盒子自动生成的 `pack.inp` 改一行——把两块板从「随机填」改成「固定在上下两端」，再回到界面继续跑。搭好的体系正是姊妹篇 [LAMMPS机械控压](C02-LAMMPS机械控压.md) 的输入。

**本文讲**：怎么用 MDStudio 备好组分、装盒，并通过手改 `pack.inp` 把两板固定成三明治。**不讲**：为什么界面体系不能用全局 barostat、怎么施加活塞压力（都在 [LAMMPS机械控压](C02-LAMMPS机械控压.md)）；力场本身的物理（见 [经典全原子力场](../00-知识文档/K03-经典全原子力场.md)）。

**前置**：会用 MDStudio 的基本操作（结构/力场、文件工作区），见 [MDStudio力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md) 与 [资源管理器](../../在线工具/00-MDStudio/M04-MDStudio资源管理器.md)；手上有能跑 LAMMPS 的环境（生产长跑用）。

## 一、体系与整体思路

我们要搭的目标体系长这样：

| 部件       | 角色        | 本例用什么                             |
| -------- | --------- | --------------------------------- |
| **上板**   | 固体基质（同质墙） | 单一元素的片状晶格（如石墨烯片，原子类型 `Cw`）        |
| **下板**   | 固体基质（异质墙） | 两种元素混排的片（原子类型 `Siw`、`Ptw`），演示异质界面 |
| **中间流体** | 受限水       | SPC/E 刚性水，约 9000 个分子              |

三明治沿 **z 方向**叠：下板在底、上板在顶，水填满中间的缝。x、y 方向做周期（板在平面内无限延展的近似），z 方向先留足够高度。

关键在于：**搭建盒子默认是「均匀装盒」**——它给每个物种写的都是 `inside box`，让分子在整个盒子里随机摆放。这对体相混合物正好，但对我们这个三明治不行：板会被当成普通分子随机丢进盒子，而不是规规矩矩贴在上下两端。所以整条流程只有一处要动手：**把 `pack.inp` 里两块板的 `inside box` 改成 `fixed`**，钉死在 z 的上下两端；水仍旧随机填中间。除此之外全在 MDStudio 界面里点。

> **为什么板要单独当「墙」？** 板在模拟里一般不参与成键、也不需要内部动力学（我们只关心它对水的约束和界面作用）。把它当成一组带 LJ/电荷的「墙原子」，力场最简单，后面还能整体固定或当活塞用。

## 二、在 MDStudio 备好组分（结构 + `.ff`）

搭建盒子只认「已声明并能找到 `.ff` 的结构文件」，正常路径是先在[力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md)为每个组分得到一对 `{name}_ff.xyz` + `{name}.ff`。我们要三样东西（板可复用同一份，也可上下各一份）：

| 组分 | 怎么来 | 产物 |
|------|--------|------|
| **固体板** | [周期分子](../../在线工具/00-MDStudio/M07-MDStudio周期分子.md) 生成片状晶格（石墨烯等），或上传自备晶面切片；再走[力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md) | `wall_ff.xyz` + `wall.ff` |
| **水** | [力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md) 选 SPC/E，或从[分子仓库](../../在线工具/00-MDStudio/M08-MDStudio分子仓库.md) 导入 | `water_ff.xyz` + `water.ff` |

关于**墙力场**：固体基质当刚性墙时，`.ff` 里通常只需要非键项（LJ 的 σ、ε 与电荷），后面在 LAMMPS 里把板整体固定即可。σ、ε 决定板对水的范德华作用强弱，直接影响润湿和吸附层结构——这是界面模拟里最该认真核对的参数。生成后建议在[资源管理器](../../在线工具/00-MDStudio/M04-MDStudio资源管理器.md) 里打开 `wall.ff` 确认参数来源与取值是否符合你的体系（力场来源、σ/ε 是否与所用水模型配套，见 [经典全原子力场](../00-知识文档/K03-经典全原子力场.md)）。

> **Tips：** 异质下板（多种元素混排）在[力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md) 里会得到多种墙原子类型（`Siw`、`Ptw`……），各有自己的 σ、ε。若只想先跑通，两块板都用同质片（同一份 `wall`）最省事。

## 三、搭建盒子：① 生成 Packmol

打开[搭建盒子](../../在线工具/00-MDStudio/M11-MDStudio搭建盒子.md)，按下面设置：

1. **选物种、填个数、定顺序**：把**板**放在前、**水**放在后（顺序很重要，见下）。板个数填 `2`（上下各一），水填目标数量（如 `9000`）。
2. **盒子尺寸**：`a`、`b` 取板的横向边长，让盒子和板严丝合缝、平面内周期对齐；`c` 给足高度（比水实际需要的高，给后续控压的活塞留空间）。
3. **原点**取 `center`（盒子中心在原点，坐标范围约 `[-L/2, L/2]`），**周期边界**设 `xyz`。

点「**① 生成 Packmol**」。平台按上面的设置写出 `pack.inp` 和各组分的 `*_pack.xyz`，并把这份自动生成的 `pack.inp` 记为基线。此时 `pack.inp` 里**板和水都是 `inside box`**——板还没被固定，下一步就来改它：

```
tolerance 2.5
filetype xyz
output simbox.xyz

structure wall_pack.xyz
  number 2
  inside box -15.0 -15.0 -40.0 15.0 15.0 40.0
end structure

structure water_pack.xyz
  number 9000
  inside box -15.0 -15.0 -40.0 15.0 15.0 40.0
end structure
```

## 四、手改 `pack.inp`：把两板钉在上下两端

这是全流程唯一的手动步骤。到[资源管理器](../../在线工具/00-MDStudio/M04-MDStudio资源管理器.md) 打开 `pack.inp`，把**板那一段**从「一份 `number 2` 的随机填」拆成**两份 `number 1` 的固定块**，分别钉在 z 的正负两端；水保持 `inside box`，但把它的 z 范围收进两板之间：

```
tolerance 2.5
filetype xyz
output simbox.xyz

# 上板：固定在 z = +38.225，姿态不旋转
structure wall_pack.xyz
  number 1
  center
  fixed 0. 0. 38.225 0 0 0
end structure

# 下板：固定在 z = -38.225
structure wall_pack.xyz
  number 1
  center
  fixed 0. 0. -38.225 0 0 0
end structure

# 水：只在两板之间随机填（z 收窄到板内侧）
structure water_pack.xyz
  number 9000
  inside box -15.0 -15.0 -38.0 15.0 15.0 38.0
end structure
```

要点：

- **`center` + `fixed x y z 0 0 0`**：以板质心为基准，把它钉到 `(0, 0, ±d/2)`、姿态不动（后三个 `0` 是不绕 x/y/z 转）。两块板一上一下就把水夹住了。`±d/2` 取略小于半盒高的值，给板和盒壁留一点余量。
- **水的 z 范围收窄**到两板内侧（这里 `±38`），避免水被填到板外面。
- **顺序不变**：两块板的固定块仍排在水前面，和搭建盒子表单里「板在前、水在后」一致——③ 生成 LAMMPS 时平台按物种顺序从 `simbox.xyz` 切分坐标，顺序错了拓扑就会套错（详见[搭建盒子](../../在线工具/00-MDStudio/M11-MDStudio搭建盒子.md) 第五节）。
- **总数不变**：板还是 2 份、水还是 9000，和表单一致；只是把「随机」改成了「固定」。

> **覆盖保护：** 平台会把 `pack.inp` 和自动生成的基线对比。检测到你手改过后，界面会提示**直接点「② 运行 Packmol」用当前文件**即可——**不要再点「① 生成 Packmol」**，否则会用表单重新覆盖你的手改版（真要重生成得先勾选「覆盖手动编辑的 pack.inp」）。重要脚本先在[资源管理器](../../在线工具/00-MDStudio/M04-MDStudio资源管理器.md) 里复制一份备份。

## 五、② 运行 Packmol、③ 生成 LAMMPS

- 点「**② 运行 Packmol**」：平台用你手改的 `pack.inp` 调 Packmol，把两板固定、水填满中间，得到 `simbox.xyz`。在可视化区看一眼——上下两板应整齐贴在盒子两端，水充满缝隙、没有明显穿插或大空洞。
- 点「**③ 生成 LAMMPS**」：平台校验 `simbox.xyz` 原子数与「物种 × 个数」一致后，写出 `data.lmp`、`in.lmp`（type 多时另出 `pair.lmp`）。

生成的 `data.lmp` 里原子类型大致是：

| type | 元素 | 属于 |
|------|------|------|
| 1 | `Cw` | 上板（同质墙） |
| 2 / 3 | `Siw` / `Ptw` | 下板（异质墙） |
| 4 | `Hw` | 水的 H |
| 5 | `Ow` | 水的 O |

到这里，一个 3 万原子级别（9000 水 + 上下板）的「板—水—板」体系就齐了，可以下载 `data.lmp` / `in.lmp` 备份。

> **顺序与个数别再动。** 从 ① 到 ③ 之间（尤其手改 `pack.inp` 后）不要改物种顺序或个数：②/③ 只校验总原子数，不查坐标—拓扑的对应关系，顺序错了会生成「看着正常、实则拓扑错乱」的 `data.lmp`。要改就从 ①（勾选覆盖）重来。

## 六、初跑一小段，确认体系能动

先做一次 [测试模拟（LAMMPS 冒烟）](../../在线工具/00-MDStudio/M12-MDStudio测试模拟.md)：只为确认体系合法、读得进 `data.lmp`、没有原子重叠飞出，不是正式生产。

**但注意**：搭建盒子默认的 `in.lmp` 用的是**全局 `fix npt`**。对眼下这种有界面的三明治体系，全局 barostat 会去缩放整个盒子（包括固体板），并不能在水的**体相区**维持你想要的压力——这正是下一篇要解决的问题。所以这里跑通即可，别把它当正式平衡，正式生产请把体系下载到计算节点，按 [LAMMPS机械控压](C02-LAMMPS机械控压.md) 的方式改控压再长跑。

## 常见问题

**Q：为什么不能直接用搭建盒子，非要手改 `pack.inp`？**
A：搭建盒子给每个物种写的是 `inside box`（整盒随机），会把板也当普通分子随机丢。要做「板固定在两端」的三明治，必须把板的段改成 `fixed`——这是 Packmol 才有的约束，表单目前不直接暴露，所以手改 `pack.inp` 最直接。

**Q：改完 `pack.inp` 点了 ① 结果被重置了？**
A：① 会用表单重新生成并覆盖手改版。改完应**直接点 ②**；确需重新生成再勾选「覆盖手动编辑的 pack.inp」，并提前备份手改脚本。

**Q：水钻进板里，或者板附近有大空洞？**
A：多半是水的 `inside box` 的 z 范围超出了两板之间。把水的 z 上下限收进 `fixed` 位置以内（留一点余量），并适当调大 Packmol 容差。

**Q：一定要用石墨烯当板吗？上下必须不同材料吗？**
A：都不必。任何能生成/上传成「片状、横向尺寸和盒子一致」的固体基质都行；上下同质（两份一样的 `wall`）最省事，本文用同质上板 + 异质下板只是顺便演示异质界面。

**Q：为什么盒子 z 方向要留那么高？**
A：为下一篇的机械控压活塞预留移动空间；带电体系需要三向周期（PPPM 静电），所以用「加高盒子」而不是开非周期边界来腾地方。

## 小结

- 界面/受限体系要搭「板—水—板」三明治：**板固定、水填缝**，别指望丢进周期盒子跑 NPT 了事。
- 结构与力场全在 MDStudio 备齐：板用[周期分子](../../在线工具/00-MDStudio/M07-MDStudio周期分子.md) / 自备结构 + [力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md)，水用[力场生成](../../在线工具/00-MDStudio/M09-MDStudio力场生成.md) / [分子仓库](../../在线工具/00-MDStudio/M08-MDStudio分子仓库.md)；每个组分都要有配套 `.ff`。
- 搭建盒子默认均匀装盒（`inside box`）；三明治的关键是**手改 `pack.inp`**：把两块板改成 `center` + `fixed`，钉在 z 上下两端，水收窄到板内侧。
- 手改后**直接点 ② 用当前文件**，别再点 ① 覆盖；保持物种顺序（板在前、水在后）与个数不变。
- 先冒烟测试确认能动，但全局 NPT 不适合界面体系，正式控压见下一篇。
