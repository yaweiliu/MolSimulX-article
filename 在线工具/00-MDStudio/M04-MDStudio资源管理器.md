> **系列标签：** `MDStudio` · `工作区` · `文件`

力场跑完了却找不到 `.ff`、改过的 `pack.inp` 被一键重新生成盖掉、工作区「满了」无法上传——这些问题多半出在**左侧资源管理器（工作区）**，而不是力场算法本身。

本文专讲工作区：界面结构、上传下载、文件夹、删除与清空、文件分组、选择方式，以及**单击 / 双击**各自会触发什么（含 `.mol` 单击载入「孤立分子」画板）。

![mdstudio-explorer](../../images/articles/在线工具/M04-MDStudio资源管理器/web/M04-hero-mdstudio-explorer.webp)

---

## 一、工作区是什么？

可以把它想成浏览器里的**当前项目文件夹**：各 Tab 生成的结构、力场、`pack.inp`、`simbox.xyz`、`data.lmp`、`in.lmp` 与任务输出，默认都落在这里。换设备登录同一账号时，以产品实际同步策略为准；习惯上把最终输入**下载备份**到本机。

产品文案可能写「资源管理器」或「工作区」——二者指同一左侧面板。中间各 Tab 的文件下拉框、右侧 3D 预览，都会跟这里保持同步。

---

## 二、界面组成

自上而下大致分为四块：

| 区域 | 作用 |
|------|------|
| **上传区** | 点击选择本地文件，支持一次选多个；上传目标为**当前所在目录** |
| **路径栏** | 显示 `/`（根目录）或 `/文件夹名`；进入子文件夹后会出现 |
| **文件列表** | 文件夹、文件、扩展名标签、大小；不同文件组之间用细线分隔 |
| **底部提示条** | 操作成功或失败时短暂显示 toast（约数秒后自动消失） |

在列表空白处或条目上**右键**，可打开上下文菜单；重命名、复制、移动、删除确认等会以浮动面板形式出现，不遮挡整页。

上传区下方有一行灰色说明，概括当前档位的**单文件上传上限、工作区总容量、每目录条目上限**（具体数字以界面为准，VIP 与免费对照见 [使用须知](M02-MDStudio使用须知与限制.md)）。

---

## 三、文件夹与目录结构

工作区采用**单层文件夹**设计，便于在浏览器里管理，也避免路径过深：

| 规则 | 说明 |
|------|------|
| **根目录 `/`** | 可同时放**文件**和**子文件夹**；仅在此处可「新建文件夹」 |
| **子文件夹内** | 只能放**文件**，不能再建子文件夹（不支持嵌套目录） |
| **路径深度** | 任意文件的路径最多为 `文件夹/文件名`，不会出现 `a/b/c` |
| **返回上级** | 进入子文件夹后，列表首行会出现 `..`；**双击** `..` 回到根目录 |
| **文件夹名** | 不能含 `/`、`\`、`.` 开头等非法字符；不能与已有条目重名 |

**单击**文件夹：仅选中（高亮），不进入。**双击**文件夹（含普通文件夹与 `..`）：进入该文件夹或返回上级。

子文件夹适合把「多组分配方」「某次试算的中间文件」与根目录默认流水线产物分开；各 Tab 的文件选择器会识别**当前所在目录**，只列出该目录下符合条件的文件。

## 四、上传

| 项 | 说明 |
|----|------|
| **权限** | **上传需要 VIP**；免费档请用「孤立分子」画板生成结构，或走不依赖上传的主流程（见 [使用须知](M02-MDStudio使用须知与限制.md)） |
| **方式** | 点击上传区选择文件；支持多选 |
| **目标位置** | 上传到**当前路径**：在根目录则落根目录，在子文件夹则落该文件夹 |
| **同名文件** | **不会覆盖**已有文件；同名项会被跳过，并在 toast 中说明 |
| **大小与配额** | 单文件不能超过上传上限；所有文件合计不能超过工作区总容量；当前目录条目数已满时，上传区会禁用并提示先删除 |
| **典型用途** | 本地改过的 `pack.inp`、`simbox.xyz`；LigParGen / ATB 的 `.gro`+`.itp` 或 `.pdb`+`.itp`；电荷续跑用的 `.out`；本机 Packmol 跑完回传的 `simbox.xyz` 等 |

上传完成后列表自动刷新；若上传的是可预览的结构文件，右侧 3D 可能自动切换到新文件。

## 五、下载

通过右键菜单中的下载项（文案会随选中状态变化）：

| 场景 | 行为 |
|------|------|
| **选中 1 个普通文件** | 浏览器直接下载该文件 |
| **选中 1 个文件夹** | 打包为 `.zip` 下载（含文件夹内全部内容） |
| **选中多个文件 / 文件夹** | 打包为 `selection.zip` |
| **未选中任何项，且在子文件夹内** | 打包下载**当前文件夹**全部内容（`文件夹名.zip`） |
| **未选中任何项，且在根目录** | 打包下载**整个工作区**（`workspace.zip`） |

建议至少备份：`data.lmp`、`in.lmp`、关键 `.ff`、装盒用的 `simbox.xyz` 或 `*_ff.xyz`，以及你手改过的 `pack.inp`。

## 六、删除与清空

| 操作 | 说明 |
|------|------|
| **删除单个文件** | 选中后点删除，**立即执行**（无二次确认） |
| **删除多项** | 选中多个文件或文件夹后删除，会弹出确认 |
| **清空当前范围** | 未选中任何项时，删除菜单变为「清空工作区」或「清空当前文件夹」；会删除该范围内**全部内容（含隐藏文件）**，不可恢复，需确认 |

删除文件夹会连同其内文件一并移除。清理时优先删：失败任务的中间产物、重复 `simbox`、旧日志、不再需要的 `*_pack.xyz`。

删除 **`.cif`** 时，系统会顺带删除同目录下与之关联的**隐藏结构缓存**（`.{cif名}.msx_cif_cache.mol2` / `.json`），避免留下孤儿文件。可见的 `*_p1.cif`、`*_unwrap.cif` 不会自动删，需自行处理。

## 七、重命名、复制、移动

以上操作均需**先选中条目**，且多数通过右键菜单触发。

### 重命名

- 每次只能重命名 **1 项**（文件或文件夹）。
- 对带扩展名的文件，输入框默认**只选中主文件名（stem）**，扩展名保留，避免误改 `.ff`、`.lmp` 等后缀。
- 不能与已有条目重名。

### 复制

- 每次只能复制 **1 项**。
- 默认新名为 `原名_copy`；若已存在则自动尝试 `原名_copy1`、`原名_copy2`……
- 复制文件夹会在根目录生成新的同级文件夹（受根目录条目上限约束）。

### 移动

- 仅支持**文件**，**不支持移动文件夹**。
- 可选目标：工作区根目录 `/`，或与其他源文件**共同的**其它一级文件夹。
- 多选时，只有所有选中文件都能到达的目标才会出现在列表中。

## 八、文件分组与排序

列表不是按纯字母顺序简单排列，而是按**流水线语义**分组，方便一眼看出「同一分子」或「同一阶段」的文件。

### 视觉分隔

- **根目录**：所有**子文件夹**排在最前；文件夹与文件之间有一条分隔线。
- **文件之间**：每当「分组键」变化时，插入一条细分隔线（同一组内无分隔线）。

### 分组逻辑（概念）

1. **按分子 / 流水线归组**  
   系统会根据文件名剥掉常见后缀（如 `_ff.xyz`、`_pack.xyz`、`_trans.xyz`、`.am1-bcc.out` 等），把 `EtOH.mol`、`EtOH_ff.xyz`、`EtOH.ff` 等归到同一组 **EtOH**。LigParGen 的 `.gro` 与 `.itp` 也会按相同 stem 归组。

2. **工作区级文件单独成组**（排在分子组之后），内部再分子类：
   - **装盒核心**：`pack.inp`、`simbox.xyz`
   - **脚本**：`run_packmol.sh`、`run_cp2k.sh`、`run_ambertools.sh` 等
   - **Lammps 输入**：`data.lmp`、`in.lmp`、`pair.lmp`、`in.test.lmp`（组内按此顺序）
   - **测试结果**：`result_thermo.log`、`result_atoms.xyz`、轨迹文件等

3. **组内顺序**  
   同一分子组内，大致按流水线先后排列：结构源文件 → 电荷相关 → `_ff.mol2` / `_ff.xyz` → `.ff` → 装盒坐标 → 其它输出。

分组只影响**显示顺序**，不改变磁盘上的文件名；在子文件夹内浏览时，仍按相同规则排序，但不再列出子文件夹。

## 九、选择方式

| 操作 | 效果 |
|------|------|
| **普通单击** | 清除其它选中，只选中当前行 |
| **Ctrl / ⌘ + 单击** | 切换当前行选中状态，保留其它已选项（多选） |
| **Shift + 单击** | 从上次锚点行到当前行，连续选中范围内所有**文件行** |
| **右键** | 若点在未选中行上，会先选中该行，再打开菜单 |
| **点击列表外空白** | 清除全部选中 |

文件夹与 `..` 也可被选中，用于打包下载或批量删除；但移动操作仅针对文件。

## 十、单击：预览与载入画板

单击**文件**（非文件夹）时，除选中高亮外，还会按扩展名触发联动：

| 文件类型 | 单击后 |
|----------|--------|
| **`.mol`** | 自动切换到 **「孤立分子」** Tab，并把该 `.mol` **载入 Ketcher 画板**继续编辑；分子名会尽量取自文件名。若尚未保存过其它结构，这是把已有 mol 拉回画板的最快方式 |
| **`.xyz`、`.mol2`、`.pdb`、`.zmat`、`.cif`** | 右侧 **3D 预览**切换到该文件（过大或格式不支持时可能失败，见使用须知） |
| **其它扩展名** | 仅选中，不打开预览或编辑器 |

**注意：** `.mol` 的单击行为是「送去画板」，与双击打开文本编辑器不同（见下一节）。结构预览与画板是两条路径：要看三维坐标用预览；要改二维拓扑用画板。

---

## 十一、双击：文本预览与编辑

**双击文件行**（文件夹除外）会打开**全屏文本编辑器**：

| 项 | 说明 |
|----|------|
| **适用** | `pack.inp`、`in.lmp`、`.ff`、日志、脚本等文本类文件 |
| **可编辑上限** | 超过「可编辑大小」的文件以**只读**打开，可查看但不能保存 |
| **预览上限** | 超过「预览大小」的文件无法打开编辑器 |
| **保存** | 可编辑文件支持保存、另存为；关闭时有未保存修改时会提示 |
| **搜索** | 编辑器内支持查找（Ctrl/⌘+F、F3 跳转匹配） |

双击会取消单击的选中手势，避免误触；因此 **`.mol` 若要进画板请用单击，若要查看/改 molfile 文本请用双击**。

---

## 十二、右键菜单一览

在文件列表内右键，菜单项随选中状态变化，常见包括：

| 菜单项 | 条件 / 说明 |
|--------|-------------|
| **刷新** | 始终可用；任务结束后若未见新文件可手动刷新 |
| **新建文件夹** | 仅在根目录且未达条目上限时 |
| **重命名 / 复制** | 恰好选中 1 项 |
| **下载** | 见第五节 |
| **移动** | 选中至少 1 个文件，且未选文件夹 |
| **删除** | 有选中则删选中项；无选中则变为清空当前范围 |
| **清空工作区 / 清空当前文件夹** | 当前目录非空且未选中项时出现 |

列表滚动时菜单会自动关闭。

---

## 十三、与各 Tab 的联动

工作区不是孤立文件柜，而是整条流水线的枢纽：

- **切换 Tab 时**，各功能页里的文件下拉框会带上当前 `dir` 参数，只显示**当前文件夹**内符合条件的文件（例如装盒只列含 `.ff` 引用的物种坐标）。
- **生成新文件**（力场、装盒、测试等）后，列表会刷新；结构类输出常会自动聚焦到右侧 3D。
- **孤立分子**保存 `.mol`、**周期分子**导出 `.cif` 等，都会直接写入当前工作区路径。
- 重量级任务进行时，对应 Tab 内仍有**实时任务日志**；部分任务还会在 workspace 留下 `packmol.log`、`result_thermo.log` 等文件，可在工作区双击查看，或按分组落在「测试结果」一组。

## 十四、手改 `pack.inp` 再回传

这是上传 + 编辑的组合用法，也是资源管理器最高频场景之一。

**重要：** 「搭建盒子」里运行 Packmol 时，**始终读取当前目录下的 `pack.inp`**，Tab 内**没有**「另选其它 `.inp`」的入口。因此手改结果必须最终仍落在名为 `pack.inp` 的文件上。

推荐流程：

1. 在「搭建盒子」生成一版 `pack.inp`，必要时先**下载备份**。
2. **在线改：** 在资源管理器中**双击** `pack.inp`，改完直接保存。  
   **本地改：** 下载到本机修改分子数、盒子边长、约束等。
3. 本地改完后需回传：上传**不会覆盖**同名文件，应先**删除**工作区里的旧 `pack.inp`，再上传改好的同名文件。  
   若曾另存为其它文件名（如 `pack_custom.inp`），请再**重命名回 `pack.inp`**（或删掉旧 `pack.inp` 后把新文件改名为 `pack.inp`），否则 Packmol 仍会读原来的 `pack.inp`。
4. 系统若检测到 `pack.inp` 曾被手改，装盒 Tab 会提示「已手动编辑」；重新点「生成 Packmol 输入」前需勾选确认覆盖。也可跳过重新生成，直接点 ② 使用当前 `pack.inp` 跑 Packmol。
5. 在线保存 `pack.inp` 后，上述提示状态会同步更新。

更大、更定制的改法见 [搭建模拟盒子（Packmol 三步）](M11-MDStudio搭建盒子.md)。

## 十五、空间与条目不够时

1. **删**：中间产物、失败半成品、重复日志、旧 `*_pack.xyz`。
2. **下**：把要留存的 `data.lmp`、`in.lmp`、`.ff` 等下载到本机。
3. **清**：确认无需保留后，用「清空工作区」一次性释放空间（含隐藏文件）。
4. **对照限额**：工作区总容量、单文件上传/编辑/预览上限、每目录条目数等，以界面上传区提示与 [使用须知](M02-MDStudio使用须知与限制.md) 为准。

> **Tips：** 列表中**不显示**以 `.` 开头的隐藏文件。它们**不计入**每目录条目上限与工作区容量配额；普通删除选不中它们，但「清空」会一并删掉。若条目数触顶，需删可见文件（或清空）后才能继续上传或新建文件夹。

---

## 小结

1. 工作区即当前项目目录；根目录可建**一层**子文件夹；`..` 与文件夹**双击**进入，**单击**只选中。
2. **上传**需 VIP，支持多文件，不覆盖同名；**下载**支持单文件、多选 zip、整夹或整区打包。
3. 文件按**分子 / 流水线**分组排序；单击 `.mol` 进**孤立分子**画板，单击结构文件开 **3D**，双击开**文本编辑器**。
4. 支持多选、重命名、复制、移动（仅文件）、删除与清空；右键菜单与 toast 是主要操作入口。
5. 与各 Tab 文件选择器、右侧预览联动；手改 `pack.inp` 后须仍名为 `pack.inp`（Tab 不能另选其它 `.inp`），注意覆盖提示与备份。
6. 配额与上传权限见 [使用须知](M02-MDStudio使用须知与限制.md)。
